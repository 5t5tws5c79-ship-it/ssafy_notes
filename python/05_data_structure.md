# Python 데이터 구조 정리 (Data Structure)

## 1. 데이터 구조란?

여러 데이터를 **효과적으로 사용·관리하기 위한 구조**. `str`, `list`, `dict` 등이 여기 해당한다.

- CS(컴퓨터 과학)에서는 **자료 구조**라고 부른다.
- 각 데이터를 **효율적으로 관리**하기 위한 구조를 나누어 놓은 것.
- 단순히 데이터를 묶는 것을 넘어, **프로그램의 성능·효율성·유지보수성**에 큰 영향을 미치는 핵심 개념이다.

---

## 2. 메서드 (Method)

**메서드란?** **객체에 속한 함수.** 함수처럼 **호출**해서 쓴다.

- 문자열, 리스트, 딕셔너리 등 다양한 데이터 구조는 저마다 **고유한 메서드**를 가진다.
- 메서드는 **클래스 내부에 정의되는 함수**다.
- **클래스**는 파이썬에서 **타입을 표현하는 방법**이며, 지금까지도 은연중에 계속 사용해온 개념이다. (자세한 내용은 후반부 클래스 챕터에서 다룸)

### 메서드 활용 예시

```python
'hello'.capitalize()   # 문자열 메서드

my_list = [1, 2, 3]
my_list.append(4)      # 리스트 메서드
```

## 3. 공통 시퀀스 메서드 (Common Sequence Methods)

모든 시퀀스(str, list, tuple 등)에서 공통으로 쓸 수 있는 메서드.

| 메서드 | 설명 |
|--------|------|
| `s.index(x)` | 시퀀스 `s`에서 **첫 번째로 일치하는** 항목 `x`의 인덱스를 반환. 없으면 **ValueError** 발생 |
| `s.count(x)` | 시퀀스에서 등장하는 `x`의 **개수**를 반환 |

```python
s = 'hello world'
s.index('o')     # 4  (첫 번째 'o'의 위치)
s.count('o')     # 2
```

## 4. 불변 시퀀스 메서드 — 탐색 및 검증 (str 전용)

### `find()` — 위치 찾기 (에러 없이)

시퀀스에서 첫 번째 위치를 반환. **없으면 -1을 반환.**

- `x`의 **위치를 알아야 할 경우** 사용한다. (에러를 피하고 싶을 때 `index()` 대신 사용)

```python
s = 'hello'
s.find('l')     # 2
s.find('z')     # -1  (없으면 에러 대신 -1)
```

> **`find()` vs `index()`:** 둘 다 위치를 찾지만, 못 찾았을 때 `find()`는 `-1`을, `index()`는 **ValueError**를 발생시킨다. "없을 수도 있는" 상황이면 `find()`, "반드시 있어야 하는" 상황이면 `index()`가 적합하다.

### `isupper()` / `islower()` — 대소문자 검증

문자열 내의 **모든 케이스 문자**(대소문자 구분이 있는 문자)가 대문자/소문자로만 이루어져 있는지 확인.

```python
'HELLO'.isupper()    # True
'Hello'.isupper()    # False  (섞여 있음)
'hello'.islower()    # True
'HELLO123'.isupper() # True   (숫자는 케이스가 없으므로 무시됨)
```

> 숫자·공백·특수문자처럼 **대소문자 구분이 없는 문자**는 검사에서 무시된다. 단, 케이스가 있는 문자가 **하나도 없으면** `False`를 반환한다.

### `isalpha()` — 알파벳(문자) 검증

문자열이 **비어 있지 않고**, **모든 글자가 문자(alphabetic)일 때만** `True`를 반환.

- 여기서 "문자"는 **유니코드상 문자로 분류되는 모든 글자**를 뜻한다. (한글도 포함)
- 숫자, 공백, 특수문자가 **하나라도 섞이면** `False`.

```python
'hello'.isalpha()     # True
'hello123'.isalpha()  # False  (숫자 섞임)
'hello world'.isalpha() # False  (공백 섞임)
'안녕'.isalpha()       # True   (한글도 문자로 인정)
''.isalpha()           # False  (빈 문자열은 False)
```

## 5. 문자열 조작 메서드 (String Manipulation)

**핵심 원칙:** 문자열은 **불변 시퀀스**이기 때문에, 아래 메서드들은 원본을 바꾸지 않고 **모두 새로운 문자열을 반환**한다.

### ⭐ 반드시 알아야 하는 메서드

| 메서드 | 설명 |
|--------|------|
| `str.replace(old, new[, count])` | 기존 문자열에서 `"old"`라는 부분 문자열이 `"new"`로 **모두** 바뀐 문자열을 반환. `[count]`는 **선택 인자** (대괄호 표시) |
| `str.strip([chars])` | **선행과 후행**의 문자가 제거된 문자열의 **복사본**을 돌려줌 |
| `str.split(sep=None, maxsplit=-1)` | `sep`을 구분자 문자열로 사용해 문자열에 있는 단어들의 **리스트**를 반환 |
| `str.join(iterable)` | 구분자로 `iterable`의 문자열들을 **연결**한 문자열을 반환 |

#### `replace()` 상세

```python
text = 'hello world'
text.replace('o', '0')          # 'hell0 w0rld'  (모든 'o'가 바뀜)
text.replace('o', '0', 1)       # 'hell0 world'  (count=1 → 첫 1개만)
```

#### `strip()` 상세

- `chars` 인자는 **제거할 문자 집합.** 생략되거나 `None`이면 **공백**을 제거한다.
- **중간에 있는 문자는 제거하지 않는다** — 오직 **양쪽 끝**만.
- 제거할 문자 집합 중 **하나라도 양쪽 끝에 있으면 계속해서 제거**한다.

```python
'  hello  '.strip()          # 'hello'          (양쪽 공백 제거)
'xxhelloxx'.strip('x')       # 'hello'          ('x'가 양쪽 끝에 있는 만큼 제거)
'xyhelloxy'.strip('xy')      # 'hello'          ('x'든 'y'든 끝에 있으면 계속 제거)
'hel lo'.strip()             # 'hel lo'         (중간 공백은 그대로!)
```

#### `split()` 상세

```python
'a,b,c'.split(',')           # ['a', 'b', 'c']
'a b  c'.split()             # ['a', 'b', 'c']   (기본값은 공백, 연속 공백도 하나로 처리)
'a,b,c,d'.split(',', 2)      # ['a', 'b', 'c,d'] (maxsplit=2 → 최대 2번만 분할)
```

> `maxsplit`이 주어지면 **최대 `maxsplit`번의 분할**만 수행된다.

#### `join()` 상세

```python
words = ['hello', 'world']
' '.join(words)     # 'hello world'
'-'.join(words)      # 'hello-world'
''.join(words)       # 'helloworld'
```

> **`''`(빈 문자열)도 `str` 타입이다.** `type('')`은 `<class 'str'>`. 길이가 0인 문자열일 뿐, 여전히 문자열 객체다. 그래서 `''.join(...)`처럼 구분자 없이 이어붙일 때도 문제없이 동작한다.

### 대소문자 변환 메서드

```python
text = 'heLLo, woRld!'

new_text1 = text.capitalize()   # Hello, world!   (첫 글자만 대문자, 나머지 소문자)
new_text2 = text.title()        # Hello, World!   (각 단어 첫 글자 대문자)
new_text3 = text.upper()        # HELLO, WORLD!   (전부 대문자)
new_text4 = text.lower()        # hello, world!   (전부 소문자)
new_text5 = text.swapcase()     # HEllO, WOrLD!   (대↔소문자 서로 교체)
```

| 메서드 | 설명 |
|--------|------|
| `str.capitalize()` | 가장 첫 번째 글자를 대문자로 변경 |
| `str.title()` | 띄어쓰기 기준으로 각 단어의 **첫 글자는 대문자**, 나머지는 소문자로 변환 |
| `str.upper()` | 모두 대문자로 변경 |
| `str.lower()` | 모두 소문자로 변경 |
| `str.swapcase()` | 대문자 ↔ 소문자를 서로 바꿈 |

## 6. 가변 시퀀스 메서드 (Mutable Sequence — 리스트)

문자열과 달리 **리스트는 가변(mutable)**이라, 아래 메서드들은 **원본 리스트 자체를 직접 수정**한다. (새 리스트를 반환하는 게 아님)

### 값 추가 및 삭제 메서드

| 메서드 | 설명 |
|--------|------|
| `L.append(x)` | 리스트 **마지막**에 항목 `x`를 추가 |
| `L.extend(iterable)` | `iterable`의 **모든 항목들**을 리스트 끝에 추가 (`+=`과 같은 기능) |
| `L.insert(i, x)` | 리스트 **인덱스 `i`**에 항목 `x`를 삽입 |
| `L.remove(x)` | 리스트에서 **가장 왼쪽(첫 번째)에 있는** 항목 `x`를 제거. 항목이 존재하지 않으면 **ValueError** |
| `L.pop()` | 리스트 **가장 오른쪽(마지막)** 항목을 **반환 후 제거** |
| `L.pop(i)` | 리스트의 **인덱스 `i`**에 있는 항목을 **반환 후 제거** |
| `L.clear()` | 리스트의 **모든 항목 제거** |

### ⚠️ `append()` vs `extend()` — 반드시 구분할 것

이 둘은 헷갈리기 쉬운 대표적인 포인트다.

```python
L1 = [1, 2, 3]
L1.append([4, 5])
print(L1)    # [1, 2, 3, [4, 5]]   ← [4, 5]가 하나의 "항목"으로 통째로 들어감

L2 = [1, 2, 3]
L2.extend([4, 5])
print(L2)    # [1, 2, 3, 4, 5]     ← [4, 5]의 각 항목이 개별로 풀려서 들어감
```

| | `append(x)` | `extend(iterable)` |
|---|---|---|
| 무엇을 넣나 | **x 하나** (통째로, 그대로) | iterable의 **각 요소를 하나씩** |
| 리스트를 넣으면 | 리스트 **자체가 항목 하나**가 됨 (중첩됨) | 리스트 **안의 값들**이 풀려서 추가됨 |
| 결과 길이 증가량 | 항상 **+1** | iterable의 **요소 개수만큼** |

```python
L = [1, 2, 3]
L.insert(1, 100)     # [1, 100, 2, 3]     인덱스 1에 100 삽입

L = [1, 2, 3, 2]
L.remove(2)           # [1, 3, 2]          왼쪽(첫 번째) 2만 제거

L = [1, 2, 3]
x = L.pop()            # x = 3, L = [1, 2]  마지막 항목 반환 후 제거

L = [1, 2, 3]
x = L.pop(0)            # x = 1, L = [2, 3]  인덱스 0 항목 반환 후 제거

L = [1, 2, 3]
L.clear()               # L = []
```

---
### 값 추가 및 삭제 메서드 — 상세 보강

#### `insert(i, x)` — 음수 인덱스 동작

`x`를 지정한 인덱스 `i` 위치에 삽입한다. **인덱스가 음수면 배열 끝에 상대적인 값**으로 처리된다.

```python
L = [1, 2, 3, 4, 5]
L.insert(-1, 100)
print(L)   # [1, 2, 3, 4, 100, 5]
```

> `-1`은 "끝에서부터 한 칸 앞으로 간 위치"에 넣는다는 뜻. 마지막 요소(`5`) **바로 앞**에 삽입되고, 마지막 요소는 그대로 뒤에 남는다. (`append`처럼 맨 끝에 붙는 게 아니라는 점에 주의)

#### `remove(x)` — 값으로 제거

리스트에서 **인자와 첫 번째로 일치하는 항목**을 삭제한다.

```python
L = [1, 2, 3, 2]
L.remove(2)
print(L)   # [1, 3, 2]   ← 첫 번째로 만난 2만 삭제, 뒤의 2는 남음
```

#### `pop()` / `pop(i)` — ⭐ 중요: "제거 + 반환"

인덱스의 항목을 **제거하고, 그 값을 '반환'**한다. 인자를 작성하지 않으면 **마지막 항목**을 제거한다.

```python
L = [10, 20, 30]

x = L.pop()       # x = 30, L = [10, 20]   (마지막 항목 제거 + 반환)
y = L.pop(0)      # y = 10, L = [20]       (인덱스 0 항목 제거 + 반환)
```

> **포인트는 "반환"이다.** `remove()`, `clear()`는 아무것도 돌려주지 않지만(`None`), `pop()`은 **원본을 조작(제거)하면서 동시에 지워진 값을 결과로 돌려준다.** 그래서 "지운 값을 바로 써야 할 때"는 `pop()`을 쓴다.

```python
stack = [1, 2, 3]
last = stack.pop()    # last=3 을 바로 활용 가능
print(f'꺼낸 값: {last}, 남은 리스트: {stack}')
# 꺼낸 값: 3, 남은 리스트: [1, 2]
```

#### `clear()` — 리스트 초기화

리스트의 **모든 항목을 제거**한다.

```python
L = [1, 2, 3]
L.clear()
print(L)    # []
```

> **포인트:** `clear()`는 리스트 **객체 자체를 없애는 게 아니라, 그 안의 요소(내용물)만 초기화**한다. 리스트라는 상자는 그대로 남고, 안이 비워지는 것.

```python
a = [1, 2, 3]
b = a          # a와 b는 같은 리스트를 가리킴 (참조 공유)
a.clear()
print(b)       # []  ← b도 함께 비워짐 (같은 객체를 가리키므로)
```

> 앞서 배운 **참조(reference)** 개념과 이어지는 부분: `a = []`처럼 **재할당**하면 `a`는 새 빈 리스트를 가리키게 되어 `b`는 영향받지 않지만, `a.clear()`는 **원본 객체 자체를 비우는 것**이라 같은 객체를 가리키는 `b`도 함께 영향을 받는다.

### 정렬 관련 메서드

| 메서드 | 설명 |
|--------|------|
| `L.reverse()` | 리스트를 **역순으로 변경**. 정렬 아님(x), 반환 없음(x) — 원본을 뒤집어버림 |
| `L.sort()` | 원본 리스트를 **오름차순으로 정렬**. `reverse=True` 인자를 주면 내림차순 |

```python
L = [3, 1, 4, 1, 5]

L.reverse()
print(L)          # [5, 1, 4, 1, 3]   ← 그냥 순서만 뒤집음 (정렬 아님!)

L = [3, 1, 4, 1, 5]
L.sort()
print(L)          # [1, 1, 3, 4, 5]   ← 오름차순 정렬

L.sort(reverse=True)
print(L)          # [5, 4, 3, 1, 1]   ← 내림차순
```

> **주의:** `reverse()`는 "정렬"이 아니라 그냥 **현재 순서를 뒤집는 것**뿐이다. 정렬된 상태로 뒤집으려면 `sort(reverse=True)`를 쓰거나 `sort()` 후 `reverse()`를 해야 한다.
> 둘 다 **원본을 직접 수정**하고 **반환값은 없다(None)**. `L = L.sort()`처럼 쓰면 `L`은 `None`이 되어버리니 주의.

## 7. 복사(Copy) 개념 정리

> 파이썬에서 **가장 헷갈리기 쉬운 주제 중 하나**라 꼼꼼히 짚고 넘어간다.

### 7-1. 가변 vs 불변 객체 — 복사를 이해하는 전제

객체 복사의 핵심을 파악하려면, 먼저 파이썬 자료구조의 **가변(mutable)**과 **불변(immutable)** 두 종류를 다시 짚어야 한다.

### 변수 할당의 진짜 의미

파이썬에서 **변수 할당은 값을 복사하는 게 아니라, 객체에 대한 "참조"를 만드는 과정**이다.

- 변수는 객체가 저장된 **메모리 주소를 가리키는 이름표(라벨)** 역할을 한다.
- 할당은 둘 중 하나다:
  1. **새로운 객체를 생성한 뒤 참조**
  2. **기존 객체에 대한 참조를 그대로 복사**

### 가변 객체 예시 — 참조를 공유하면 같이 바뀐다

```python
a = [1, 2, 3, 4]
b = a          # b는 a와 "같은 객체"를 참조 (주소를 공유)

b[0] = 100

print(a)       # [100, 2, 3, 4]  ← a도 바뀜!
print(b)       # [100, 2, 3, 4]
```

`a`와 `b`는 서로 다른 이름표지만, **같은 리스트 객체**를 가리키고 있다. 그래서 `b`를 통해 수정하면 `a`로 봐도 바뀐 게 보인다.

### 불변 객체 예시 — 재할당은 "새 객체"를 만든다

```python
a = 20
b = a          # b도 a와 같은 20 객체를 참조

b = 10         # b에 "재할당" → 새로운 객체 10을 만들어 b가 그걸 가리킴

print(a)       # 20   (그대로)
print(b)       # 10   (새 객체를 가리키게 됨)
```

`b = 10`은 기존 `20` 객체를 **수정한 게 아니라**, 새로운 `10`이라는 객체를 만들어서 `b`의 참조 대상을 **바꿔치기**한 것이다. `a`는 여전히 원래 `20`을 가리키므로 영향이 없다.

> **핵심 차이:**
> - 가변 객체(`b[0] = 100`) → **객체 내용을 직접 수정** → 참조를 공유하는 모두가 영향받음
> - 불변 객체(`b = 10`) → **새 객체를 만들어 참조 대상을 교체** → 원래 참조하던 변수(`a`)는 영향 없음

### 가변·불변의 메모리 관리 방식

| 구분 | 동작 |
|------|------|
| **가변 객체** | 생성 후에도 **내용을 수정**할 수 있음. 내용이 바뀌어도 **같은 메모리 주소를 유지** |
| **불변 객체** | 생성 후 **값을 변경할 수 없음**. 새 값을 할당하면 **새 객체가 생성**되고, 변수는 그 새 객체를 참조하게 됨 |

### 왜 이렇게 설계했을까? — 이유

**성능 최적화**
- **가변 객체:** 내용 수정이 빈번할 때, 매번 새 객체를 생성하는 대신 **기존 객체를 직접 수정**할 수 있다. 객체 생성·삭제에 드는 비용을 절감해 성능을 높인다.
- **불변 객체:** 값이 변경 불가능하므로, **여러 변수가 동일한 객체를 안전하게 공유**할 수 있다. (누군가 몰래 값을 바꿀 걱정이 없음)

## 8. 얕은 복사 (Shallow Copy)

**얕은 복사란?** 객체의 **최상위 요소만** 새로운 메모리에 복사하는 방법.

- 내부에 **중첩된 객체**가 있다면, 그 객체는 복사되지 않고 **참조(주소)만 복사**된다.
- 즉, 겉 상자는 새로 만들지만, 그 안에 담긴 것들은 **원본과 여전히 같은 것**을 가리킨다.

### ⚠️ 얕은 복사의 함정 — 중첩된 가변 객체

얕은 복사 후, **중첩된 리스트나 딕셔너리 같은 가변 객체를 수정하면 원본과 복사본이 함께 변경된다.**
→ 왜? **중첩 객체를 (복사한 게 아니라) 참조하고 있기 때문.**

### 얕은 복사를 만드는 3가지 방법

```python
a = [1, 2, [3, 4, 5]]

# 방법 1) 슬라이싱
b = a[:]

# 방법 2) list() 함수
b = list(a)

# 방법 3) copy 모듈의 copy()
import copy
b = copy.copy(a)
```

> 셋 다 **얕은 복사**다. `b = a[:]`를 하면 "새로운 리스트에 복사된 것은 **값이 아니라 (내부 요소의) 주소**"라는 점이 핵심.

### 예시로 확인하기

```python
a = [1, 2, [3, 4, 5]]
b = a[:]              # 얕은 복사

# --- 최상위 요소 수정 ---
b[0] = 999
print(a)    # [1, 2, [3, 4, 5]]      ← a는 안 바뀜!
print(b)    # [999, 2, [3, 4, 5]]

# --- 중첩된(내부) 요소 수정 ---
b[2][1] = 100
print(a)    # [1, 2, [3, 100, 5]]    ← a도 바뀜! (함정)
print(b)    # [999, 2, [3, 100, 5]]

print(f'a[2]와 b[2]가 같은 객체인가? {a[2] is b[2]}')   # True
```

**왜 이런 차이가 날까?**

| 조작 | 결과 | 이유 |
|------|------|------|
| `b[0] = 999` | `a`는 영향 없음 | `b[0]`은 최상위 요소. 얕은 복사가 **새로 복사한 부분**이라 독립적 |
| `b[2][1] = 100` | `a`도 함께 바뀜 | `b[2]`는 **내부 리스트의 참조**일 뿐, `a[2]`와 `b[2]`는 **같은 객체**(`a[2] is b[2]` → `True`)를 가리킴 |

> **한 줄 정리:** 얕은 복사는 "**바깥 상자만 새로 만들고, 안에 든 상자는 원본과 공유**"한다. 안쪽 상자(중첩된 가변 객체)를 건드리면 원본도 함께 바뀐다.

### 얕은 복사의 한계 — 2차원 리스트

2차원 리스트처럼 **mutable 안에 mutable이 있는 경우**, 얕은 복사만으로는 완전히 독립적인 복사본을 만들 수 없다. 위 예시가 바로 그 한계를 보여준다.

## 9. 깊은 복사 (Deep Copy)

**깊은 복사란?** 객체의 **모든 수준(레벨)의 요소**를 새로운 메모리에 복사하는 방법.

- **중첩된 객체까지 모두** 새로운 객체로 생성한다.
- 그래서 복사본을 아무리 깊이 수정해도 **원본에 절대 영향을 주지 않는다.**

### `copy` 모듈 사용

```python
import copy

original = [1, 2, [3, 4, 5]]
deep_copied = copy.deepcopy(original)

deep_copied[2][1] = 999

print(original)      # [1, 2, [3, 4, 5]]     ← 완전히 안전!
print(deep_copied)   # [1, 2, [3, 999, 5]]
```

### 얕은 복사 vs 깊은 복사 — 최종 정리

| | **얕은 복사 (Shallow Copy)** | **깊은 복사 (Deep Copy)** |
|---|---|---|
| 방법 | `a[:]`, `list(a)`, `copy.copy(a)` | `copy.deepcopy(a)` |
| 복사 범위 | **최상위 요소만** 새로 복사 | **모든 중첩 요소까지** 새로 복사 |
| 중첩 객체 | 원본과 **참조를 공유** (연결됨) | 원본과 **완전히 독립** |
| 중첩 요소 수정 시 | 원본도 **함께 바뀜** | 원본은 **영향 없음** |
| 언제 쓰나 | 1차원 자료구조, 또는 중첩 데이터를 공유해도 될 때 | 2차원 이상 중첩 구조를 **완전히 독립**시켜야 할 때 |

## 10. 리스트 컴프리헨션 (List Comprehension)

**간결하고 효율적으로 리스트를 생성하는 방법.** "파이써닉(Pythonic)"한 코드의 대표 사례.

```python
[표현식 for 요소 in 이터러블 if 조건식]
```

```python
squares = [x**2 for x in range(5)]
print(squares)     # [0, 1, 4, 9, 16]

evens = [x for x in range(10) if x % 2 == 0]
print(evens)       # [0, 2, 4, 6, 8]
```

### 2차원 배열 생성 시

```python
grid = [[0] * 5 for _ in range(5)]
# 또는
grid = [[0 for _ in range(5)] for _ in range(5)]
```

> ⚠️ `[[0] * 5] * 5`처럼 곱셈을 두 번 중첩해서 쓰면 **안 된다.** 안쪽 리스트가 전부 **같은 객체를 참조**하게 되어, 하나를 수정하면 모든 행이 같이 바뀌는 함정에 빠진다. (얕은 복사 문제와 같은 원리!) 그래서 반드시 리스트 컴프리헨션으로 **행마다 새 리스트를 개별 생성**해야 한다.

### 리스트 컴프리헨션 주의 상황

가독성이 항상 더 좋은 건 아니다. 조건이 복잡해지면 오히려 일반 `for`문이 더 읽기 좋을 수 있다.
**"Simple is better."** — 무조건 컴프리헨션을 고집하기보다 **가독성**을 우선하자.

## 11. 리스트를 만드는 3가지 방법 — 성능 비교

리스트 생성 방법: **반복문(for)**, **리스트 컴프리헨션**, **`map` 함수**

| 방법 | 성능 특징 |
|------|-----------|
| **리스트 컴프리헨션** | 가장 **파이써닉**하고, 대부분의 경우 **우수한 성능**을 보임 |
| **`map`** | `int`, `str` 등 **내장 함수와 함께 쓸 때 가장 빠름**. 람다나 사용자 정의 함수와 쓸 때는 컴프리헨션과 비슷하거나 더 느림 |
| **`for`문** | 일반적으로 **가장 느리지만**, 파이썬 버전이 올라가면서 비슷하거나 더 나은 결과를 보이기도 함. **복잡한 조건·예외 처리**가 필요할 때는 유일한 선택지이며 그 자체로 매우 유용 |

```python
# for문
result = []
for x in range(10):
    result.append(x**2)

# 리스트 컴프리헨션
result = [x**2 for x in range(10)]

# map (내장 함수 활용 시 가장 빠름)
result = list(map(str, range(10)))
```

> **정리:** 단순 변환·필터링에는 **컴프리헨션**, 내장 함수 적용에는 **`map`**, 복잡한 로직·예외 처리가 필요하면 **`for`문**을 쓰자.

## 12. 메서드 체이닝 (Method Chaining)

**여러 메서드를 연속해서 호출**하는 방식.

```python
text = '  Hello World  '
result = text.strip().lower().replace('world', 'python')
print(result)     # 'hello python'
```

- **반환값이 있는 메서드**는 그 뒤에 반환된 타입에 맞는 메서드를 또 부를 수 있다. (당연한 원리)
- **모든 메서드가 체이닝을 지원하는 것은 아니다.** 반환값이 없는(`None`을 반환하는) 메서드 뒤에 체이닝하면 에러가 난다.

```python
L = [3, 1, 2]
L.sort().reverse()   # ❌ AttributeError!
# sort()가 None을 반환하므로, None에는 reverse()가 없음
```

> 체이닝을 쓸 때는 **각 메서드가 무엇을 반환하는지**(원본을 수정하고 `None`을 반환하는지, 새 값을 반환하는지) 잘 이해하고 있어야 한다.

## 13. 문자 유형 판별 메서드

> (강사 코멘트: 시험에는 안 나온다고 함 — 참고용으로 가볍게)

| | `isdecimal()` | `isdigit()` | `isnumeric()` | 예시 |
|---|:---:|:---:|:---:|---|
| | True | True | True | `"038"`, `"٠٣٨"`(아랍 숫자), `"０３８"`(전각) |
| | False | True | True | `"⁰³⁸"`(위첨자), `"0.3.8."`, `"⓪③⑧"`(원문자) |
| | False | False | True | `"½⅕"`(분수), `"Ⅰ Ⅷ"`(로마 숫자), `"⑩⑬㊿"`, `"壹貳參"`(한자 숫자) |
| | False | False | False | `"abc"`, `"38.0"`, `"-38"` |

- **`isdecimal()`** : 순수 **10진수 숫자**만 인식 (가장 좁은 범위)
- **`isdigit()`** : `isdecimal()` + **위첨자, 원문자** 등 유니코드상 "숫자"로 분류된 것까지 인식
- **`isnumeric()`** : `isdigit()` + **분수, 로마 숫자, 한자 숫자** 등 "수치적 의미"가 있는 모든 것을 인식 (가장 넓은 범위)

> **포함 관계:** `isdecimal() ⊆ isdigit() ⊆ isnumeric()` — 즉, `isdecimal()`이 `True`면 나머지도 항상 `True`지만, 역은 성립하지 않는다.
> 셋 다 `"38.0"`, `"-38"`처럼 **소수점·부호가 붙으면 전부 False**다. (숫자처럼 보여도 "문자 유형 판별"은 순수 숫자 문자 여부만 본다.)

## 14. (참고) 파이썬 문법 규격 — EBNF

### BNF / EBNF란?

**BNF(Backus-Naur Form)**: 프로그래밍 언어의 **문법을 표현하기 위한 표기법**. 프로그래밍 언어마다 문법을 설명하는 방식이 제각각인 걸 **통일**하기 위해 쓰인다.

**EBNF(Extended BNF)**: BNF를 확장한 것. **메타 기호**를 통해 표현력을 높였다.

### 대표적인 EBNF 메타 기호

| 메타 기호 | 의미 |
|:---------:|------|
| `[ ]` | 선택적 요소 (있어도 되고 없어도 됨) |
| `{ }` | 0번 이상 반복 |
| `( )` | 그룹화 |

> 예를 들어 앞서 봤던 `str.replace(old, new[, count])`에서 `[count]`의 대괄호가 바로 이 **EBNF의 "선택적 요소"** 표기다. 공식 문서의 함수 시그니처를 읽을 때 이 기호들의 의미를 알면 훨씬 수월하다.
