# Python 데이터 구조 2 정리 (Non-sequence: Dictionary & Set)

## 1. 비시퀀스 데이터 구조란?

이번 편의 주제: **딕셔너리, defaultdict(딕셔너리의 확장), 세트, 세트 메서드, 세트의 집합 메서드, 해시 테이블**

### 왜 필요한가 — 동기 부여

> 친구 번호를 검색하기 쉬운 구조로 바꿔볼까요?

- **키-값(key-value) 구조**를 활용해 검색 속도를 높인다.
- **인덱스 대신 키**를 사용해 코드의 **가독성과 직관성**을 개선한다. (`0번째 친구` 대신 `phonebook['민수']`)
- 자료구조 선택이 **코드 품질과 유지보수성**에 직접 영향을 준다.

## 2. 시퀀스 vs 비시퀀스

| 구분 | 시퀀스 (Sequence) | 비시퀀스 (Non-sequence) |
|------|-------------------|--------------------------|
| 자료형 | `str`, `list`, `tuple`, `range` | `dict`, `set` |
| 찾는 기준 | **위치(인덱스)** | **키 또는 값 자체** |
| 접근 예시 | `books[0]` | `phonebook['민수']` / `'민수' in phonebook` |
| 슬라이싱 | 가능 | 불가 |
| 중복 | 허용 | 불가 (dict의 키, set의 요소) |

> **인덱스가 사라지면 무엇이 좋아지나요?**
> 시퀀스에서 민수의 번호를 찾으려면 리스트를 전부 뒤져(순차 탐색) 대조해야 한다. 비시퀀스는 **이름표(키)를 곧바로 인덱스처럼 써서**, 바로 그 자리로 찾아간다. 그래서 데이터가 아무리 많아져도 검색 속도가 거의 일정하게 빠르다. (뒤에서 배울 **해시 테이블** 덕분)

## 3. 딕셔너리 (Dictionary)

**키-값을 짝지어 저장**하는 자료구조. 내부적으로 **해시 테이블(Hash Table)**을 사용해 키-값 쌍을 관리한다.

- 키를 통한 값의 **삽입·삭제·검색**이 **데이터의 크기와 관계없이 매우 빠르다.**
- **키**는 **해시 가능(hashable)한 고유값**이어야 한다. (`str`, `int`, `float`, `tuple` 등 — 불변 타입)
- **값**은 중복이 가능하고, **어떤 자료형도** 저장할 수 있다.

> **참고 — 다른 언어와 비교하면?**
> 파이썬의 `dict`는 다른 언어의 `Map`, `HashMap`, 동적 객체(JS의 `object` 등)와 개념적으로 비슷하다. 공통점은 모두 **키-값 쌍을 해시 기반으로 저장**해 빠른 조회를 제공한다는 것. 세부 구현(정렬 여부, 타입 제약, 내부 구조)은 언어마다 다르지만, "키로 값을 빠르게 찾는다"는 핵심 아이디어는 동일하다.

## 4. 딕셔너리 메서드

⭐ 표시는 **꼭 알아야 하는** 메서드 (빨간색으로 강조된 것).

| 메서드 | 설명 |
|--------|------|
| ⭐ `D.get(k)` | 키 `k`에 연결된 값을 반환 (키가 없으면 **None**을 반환) |
| `D.get(k, v)` | 키 `k`에 연결된 값을 반환하거나, 키가 없으면 **기본값 v**를 반환 |
| ⭐ `D.keys()` | 딕셔너리 `D`의 **키를 모은 객체**를 반환 |
| ⭐ `D.values()` | 딕셔너리 `D`의 **값을 모은 객체**를 반환 |
| ⭐ `D.items()` | 딕셔너리 `D`의 **키/값 쌍을 모은 객체**를 반환 |
| `D.setdefault(k)` | 딕셔너리 `D`에서 키 `k`와 연결된 값을 반환 |
| `D.setdefault(k, v)` | 키 `k`와 연결된 값을 반환. **`k`가 D의 키가 아니면** 값 `v`와 연결한 키 `k`를 D에 추가하고 `v`를 반환 |
| `D.update(other)` | `other` 내 각 키에 대해, D에 있는 키면 그 값을 `other`의 값으로 **대체**. `other`에 있는데 D에 없는 키면 **추가** |
| ⭐ `D.pop(k)` | 딕셔너리 D에서 키 `k`를 **제거**하고 연결됐던 값을 **반환** (없으면 오류) |
| `D.pop(k, v)` | 키 `k`를 제거하고 값을 반환 (없으면 **v를 반환**) |
| ⭐ `D.clear()` | 딕셔너리 D의 **모든 키/값 쌍을 제거** |

### `get()` 상세 — 왜 중요한가

```python
some_dict = {'a': 1, 'b': 2}

some_dict['c']         # ❌ KeyError!  (없는 키로 직접 접근하면 에러)
some_dict.get('c')     # None          (에러 없이 None 반환)
some_dict.get('c', 0)  # 0             (기본값 지정 가능)
```

> `some_dict['some_key']`처럼 **대괄호로 직접 접근**하면 키가 없을 때 **에러가 발생할 수 있다.** 안전하게 조회하려면 `.get()`을 쓰는 게 정석.

### `keys()` / `values()` / `items()`

셋 다 **이터러블(iterable) 객체**를 반환한다. (리스트가 아니라 `dict_keys`, `dict_values`, `dict_items` 타입)

```python
d = {'a': 1, 'b': 2}

d.keys()      # dict_keys(['a', 'b'])
d.values()    # dict_values([1, 2])
d.items()     # dict_items([('a', 1), ('b', 2)])   ← 튜플로 묶여있는 것과 비슷

for k, v in d.items():
    print(k, v)     # a 1 / b 2
```

### `setdefault()`

키와 연결된 값을 반환한다. **키가 없다면**, `default`와 연결한 키를 딕셔너리에 **추가**하고 `default`를 반환한다.

```python
d = {'a': 1}

d.setdefault('a', 100)   # 'a'는 이미 있음 → 1 반환 (변화 없음)
d.setdefault('b', 100)   # 'b'는 없음 → d에 'b':100 추가하고 100 반환
print(d)                 # {'a': 1, 'b': 100}
```

### `update()`

**다른 딕셔너리(또는 키-값 쌍)** 로 딕셔너리를 갱신하고, **기존 키는 덮어쓴다.**

- 딕셔너리를 넣어도 되고, 키워드 인자를 넣어도 된다.

```python
d = {'a': 1, 'b': 2}

d.update({'b': 20, 'c': 3})   # b는 덮어씀, c는 새로 추가
print(d)     # {'a': 1, 'b': 20, 'c': 3}

d.update(d=4, e=5)            # 인자로도 가능
print(d)     # {'a': 1, 'b': 20, 'c': 3, 'd': 4, 'e': 5}
```

### `pop()`

키를 제거하고 연결된 값을 반환한다. 없으면 **에러**(또는 `default`가 있으면 `default`를 반환).

```python
d = {'a': 1, 'b': 2}

d.pop('a')          # 1 반환, d = {'b': 2}
d.pop('z')          # ❌ KeyError!
d.pop('z', '없음')   # '없음' 반환 (default 지정 시 에러 안 남)
```

> 리스트의 `pop()`과 달리, 딕셔너리는 **"마지막"이라는 개념이 없기 때문에** 반드시 **키를 인자로 넣어줘야 한다.** (`d.pop()`처럼 빈 채로 호출 불가)

### `clear()`

```python
d = {'a': 1, 'b': 2}
d.clear()
print(d)     # {}
```

## 5. 딕셔너리의 확장 — `defaultdict`

파이썬 내장 모듈 **`collections`**에서 제공하는 **딕셔너리의 확장판.**

- 딕셔너리에 **존재하지 않는 키를 조회할 때 자동으로 기본값을 생성**해주는 자료구조.
- **KeyError 걱정 없이** 값을 수정하거나 추가할 때 매우 유용하다.

### 기본 딕셔너리 vs `defaultdict` — 문자 개수 세기 비교

**상황:** 주어진 문자열에서 각 문자의 등장 횟수 세기

```python
# 기존: 기본 딕셔너리
text = 'banana'
counts = {}

for char in text:
    if char not in counts:      # 키가 있는지 매번 확인해야 함
        counts[char] = 0
    counts[char] += 1

print(counts)   # {'b': 1, 'a': 3, 'n': 2}
```

```python
# 개선: defaultdict 활용
from collections import defaultdict

text = 'banana'
counts = defaultdict(int)       # 기본값을 정수(0)로 설정

for char in text:
    # 키 확인 불필요! 없으면 0으로 자동 생성 후 +1
    counts[char] += 1

print(counts)   # defaultdict(<class 'int'>, {'b': 1, 'a': 3, 'n': 2})
```

> **핵심 차이:** 기본 딕셔너리는 매번 `if char not in counts:`로 **키 존재 여부를 직접 확인**해야 한다. `defaultdict(int)`는 없는 키를 조회하는 순간 **자동으로 `0`(int의 기본값)을 만들어주므로** 그 확인 코드가 필요 없어진다.

### `defaultdict` 활용 예제 — 색깔별 과일 분류하기

```python
from collections import defaultdict

fruits = [('red', 'apple'), ('yellow', 'banana'), ('red', 'cherry')]
fruit_by_color = defaultdict(list)     # 기본값을 빈 리스트로 설정

for color, fruit in fruits:
    # color 키가 없으면 빈 리스트 생성 -> append 바로 가능
    fruit_by_color[color].append(fruit)

print(fruit_by_color)
# defaultdict(<class 'list'>, {'red': ['apple', 'cherry'], 'yellow': ['banana']})
```

### 자주 쓰는 패턴 정리

| 패턴 | 초기화 | 용도 |
|------|--------|------|
| **숫자 세기** | `defaultdict(int)` | 키가 없으면 `0`으로 초기화 후 카운팅 |
| **그루핑 (리스트로 모으기)** | `defaultdict(list)` | 키가 없으면 **빈 리스트**로 초기화 후 `append` |

> **왜 이게 강력한가?** 그루핑 패턴은 알고리즘 문제에서 **그래프 인접 리스트 구성**이나 **분류(classification)** 작업에 특히 강력하다. 예를 들어 그래프의 간선 목록을 `{노드: [연결된 노드들]}` 형태로 만들 때, 일반 딕셔너리라면 매번 "이 노드가 이미 키로 있나?"를 확인하고 없으면 빈 리스트를 만드는 코드를 반복해야 한다. `defaultdict(list)`는 그 반복되는 방어 코드를 없애주기 때문에, 그래프 탐색(BFS/DFS) 같은 코드가 훨씬 짧고 명확해진다.

### ⚠️ 주의 — 조회만 해도 키가 생성된다

```python
d = defaultdict(int)
print(d['x'])       # 0  (조회했을 뿐인데)
print(d)             # defaultdict(<class 'int'>, {'x': 0})  ← 'x' 키가 생겨버림!
```

> 단순히 **값을 읽어보기만 해도** 그 키가 딕셔너리에 **실제로 추가**된다. `in` 연산자로 존재 여부만 확인하고 싶다면 `defaultdict`라도 `'x' in d`처럼 써야지, `d['x']`로 조회하면 의도치 않게 키가 생길 수 있으니 주의.

## 6. 세트 (Set)

세트는 **해시 테이블**을 사용해 데이터를 저장한다.

- 항목의 **고유성을 효율적으로 보장**한다.
- **추가·삭제·존재 여부 확인**이 크기에 상관없이 **매우 빠르다.**
- **수학적 집합 연산**을 간편하게 수행할 수 있는 것이 가장 큰 특징.

### 세트 메서드

| 메서드 | 설명 |
|--------|------|
| ⭐ `s.add(x)` | 세트 `s`에 항목 `x`를 추가. **이미 `x`가 있다면 변화 없음** |
| `s.update(iterable)` | 세트 `s`에 다른 iterable의 요소를 추가 |
| `s.clear()` | 세트 `s`의 모든 항목을 제거 |
| ⭐ `s.remove(x)` | 세트 `s`에서 항목 `x`를 제거. **항목 `x`가 없을 경우 KeyError** |
| `s.pop()` | 세트 `s`에서 **임의의** 항목을 반환하고, 해당 항목을 제거 |
| `s.discard(x)` | 세트 `s`에서 항목 `x`를 제거 (`remove`와 달리 **에러 없음**) |

### `add()` — 세트의 순서는 왜 랜덤일까?

```python
s = {1, 2, 3}
s.add(4)
print(s)    # {1, 2, 3, 4}  ← 순서가 항상 이 순서로 보장되는 건 아님
```

> 세트를 출력하면 순서가 뒤죽박죽인 것처럼 보여 찝찝할 수 있는데, 이건 세트가 **해시 테이블이라는 특수한 구조**로 데이터를 저장하기 때문이다. 값을 넣은 순서가 아니라 **해시값에 따라 내부적으로 배치**되므로, 리스트처럼 입력 순서를 유지하지 않는다.

### `update(iterable)`

세트에 **iterable을 풀어서** 넣는다. (리스트의 `extend`와 비슷한 개념)

```python
s = {1, 2}
s.update([3, 4, 5])
print(s)    # {1, 2, 3, 4, 5}
```

### `clear()`

```python
s = {1, 2, 3}
s.clear()
print(s)    # set()   ← 빈 세트는 반드시 set()으로 표시됨 (빈 {} 는 dict이므로)
```

### `remove(x)` vs `discard(x)` — 에러 처리 차이

```python
s = {1, 2, 3}

s.remove(1)      # OK, s = {2, 3}
s.remove(100)    # ❌ KeyError!  (항목이 없으면 에러)

s.discard(2)     # OK, s = {3}
s.discard(100)   # 에러 없음, 그냥 아무 일도 안 일어남
```

> **`remove()`가 없을 때 "KeyError"를 내는 이유:** 세트도 내부적으로는 **딕셔너리와 같은 해시 테이블 구조**를 사용한다. (세트는 "값만 있고 값(value)이 없는 딕셔너리"에 가까운 구조) 그래서 항목이 없을 때 나는 에러 이름도 `ValueError`가 아니라 딕셔너리와 똑같은 **`KeyError`**다.

### `pop()` — "임의"의 요소

```python
s = {1, 2, 3}
x = s.pop()      # 어떤 값이 나올지 예측 불가 (내부 해시 순서에 따라 결정됨)
```

> 리스트의 `pop()`은 "마지막 요소"라는 명확한 규칙이 있지만, 세트는 **순서 개념이 없기 때문에** 정말로 **임의(정해지지 않은)의 요소**가 제거·반환된다.

### ⚠️ `remove()` vs `discard()` — 어느 게 더 중요한가?

> **"조용한 실패(silent failure)" 문제 때문에 `discard()`보다 `remove()`가 더 중요하다.**

`discard()`는 항목이 없어도 에러 없이 조용히 넘어간다. 이건 편리해 보이지만, 만약 **"이 항목이 세트에 반드시 있어야 하는데 실수로 없는" 버그 상황**이라면, `discard()`는 그 사실을 **알려주지 않고 조용히 지나쳐버린다.** 반면 `remove()`는 항목이 없을 때 즉시 `KeyError`를 내서 **문제를 바로 드러낸다.** 그래서 "있는 게 확실할 때"는 `remove()`를 써서 실수를 조기에 잡아내는 게 더 안전한 습관이다.

## 7. 세트의 집합 메서드 — 수학적 집합 연산

| 메서드 | 설명 | 연산자 |
|--------|------|:------:|
| `set1.difference(set2)` | set1에는 들어있지만 set2에는 **없는** 항목으로 세트 생성 후 반환 (**차집합**) | `set1 - set2` |
| `set1.intersection(set2)` | set1과 set2 **모두** 들어있는 항목으로 세트 생성 후 반환 (**교집합**) | `set1 & set2` |
| `set1.issubset(set2)` | set1의 항목이 **모두 set2에 들어있으면** True 반환 (**부분집합**) | `set1 <= set2` |
| `set1.issuperset(set2)` | set1이 set2의 항목을 **모두 포함**하면 True 반환 (**상위집합**) | `set1 >= set2` |
| `set1.union(set2)` | set1 또는 set2에(혹은 둘 다) 들어있는 항목으로 세트 생성 후 반환 (**합집합**) | `set1 \| set2` |

```python
a = {1, 2, 3, 4}
b = {3, 4, 5, 6}

a.difference(b)      # {1, 2}       또는 a - b
a.intersection(b)     # {3, 4}       또는 a & b
a.union(b)             # {1,2,3,4,5,6}  또는 a | b

{1, 2}.issubset(a)          # True   ({1,2}는 a에 다 포함됨) 또는 {1,2} <= a
a.issuperset({1, 2})        # True   (a가 {1,2}를 다 포함함) 또는 a >= {1,2}
```

> **메서드로도, 연산자로도 쓸 수 있다.** 가독성 면에서는 연산자(`-`, `&`, `|`, `<=`, `>=`)가 더 간결하고 수학적 집합 표기와 직관적으로 일치해서 실무에서 자주 쓰인다.

## 1. 비시퀀스 데이터 구조란?

이번 편의 주제: **딕셔너리, defaultdict(딕셔너리의 확장), 세트, 세트 메서드, 세트의 집합 메서드, 해시 테이블**

### 왜 필요한가 — 동기 부여

> 친구 번호를 검색하기 쉬운 구조로 바꿔볼까요?

- **키-값(key-value) 구조**를 활용해 검색 속도를 높인다.
- **인덱스 대신 키**를 사용해 코드의 **가독성과 직관성**을 개선한다. (`0번째 친구` 대신 `phonebook['민수']`)
- 자료구조 선택이 **코드 품질과 유지보수성**에 직접 영향을 준다.

---

## 2. 시퀀스 vs 비시퀀스

| 구분 | 시퀀스 (Sequence) | 비시퀀스 (Non-sequence) |
|------|-------------------|--------------------------|
| 자료형 | `str`, `list`, `tuple`, `range` | `dict`, `set` |
| 찾는 기준 | **위치(인덱스)** | **키 또는 값 자체** |
| 접근 예시 | `books[0]` | `phonebook['민수']` / `'민수' in phonebook` |
| 슬라이싱 | 가능 | 불가 |
| 중복 | 허용 | 불가 (dict의 키, set의 요소) |

> **인덱스가 사라지면 무엇이 좋아지나요?**
> 시퀀스에서 민수의 번호를 찾으려면 리스트를 전부 뒤져(순차 탐색) 대조해야 한다. 비시퀀스는 **이름표(키)를 곧바로 인덱스처럼 써서**, 바로 그 자리로 찾아간다. 그래서 데이터가 아무리 많아져도 검색 속도가 거의 일정하게 빠르다. (뒤에서 배울 **해시 테이블** 덕분)

---

## 3. 딕셔너리 (Dictionary)

**키-값을 짝지어 저장**하는 자료구조. 내부적으로 **해시 테이블(Hash Table)**을 사용해 키-값 쌍을 관리한다.

- 키를 통한 값의 **삽입·삭제·검색**이 **데이터의 크기와 관계없이 매우 빠르다.**
- **키**는 **해시 가능(hashable)한 고유값**이어야 한다. (`str`, `int`, `float`, `tuple` 등 — 불변 타입)
- **값**은 중복이 가능하고, **어떤 자료형도** 저장할 수 있다.

> **참고 — 다른 언어와 비교하면?**
> 파이썬의 `dict`는 다른 언어의 `Map`, `HashMap`, 동적 객체(JS의 `object` 등)와 개념적으로 비슷하다. 공통점은 모두 **키-값 쌍을 해시 기반으로 저장**해 빠른 조회를 제공한다는 것. 세부 구현(정렬 여부, 타입 제약, 내부 구조)은 언어마다 다르지만, "키로 값을 빠르게 찾는다"는 핵심 아이디어는 동일하다.

---

## 4. 딕셔너리 메서드

⭐ 표시는 **꼭 알아야 하는** 메서드 (빨간색으로 강조된 것).

| 메서드 | 설명 |
|--------|------|
| ⭐ `D.get(k)` | 키 `k`에 연결된 값을 반환 (키가 없으면 **None**을 반환) |
| `D.get(k, v)` | 키 `k`에 연결된 값을 반환하거나, 키가 없으면 **기본값 v**를 반환 |
| ⭐ `D.keys()` | 딕셔너리 `D`의 **키를 모은 객체**를 반환 |
| ⭐ `D.values()` | 딕셔너리 `D`의 **값을 모은 객체**를 반환 |
| ⭐ `D.items()` | 딕셔너리 `D`의 **키/값 쌍을 모은 객체**를 반환 |
| `D.setdefault(k)` | 딕셔너리 `D`에서 키 `k`와 연결된 값을 반환 |
| `D.setdefault(k, v)` | 키 `k`와 연결된 값을 반환. **`k`가 D의 키가 아니면** 값 `v`와 연결한 키 `k`를 D에 추가하고 `v`를 반환 |
| `D.update(other)` | `other` 내 각 키에 대해, D에 있는 키면 그 값을 `other`의 값으로 **대체**. `other`에 있는데 D에 없는 키면 **추가** |
| ⭐ `D.pop(k)` | 딕셔너리 D에서 키 `k`를 **제거**하고 연결됐던 값을 **반환** (없으면 오류) |
| `D.pop(k, v)` | 키 `k`를 제거하고 값을 반환 (없으면 **v를 반환**) |
| ⭐ `D.clear()` | 딕셔너리 D의 **모든 키/값 쌍을 제거** |

### `get()` 상세 — 왜 중요한가

```python
some_dict = {'a': 1, 'b': 2}

some_dict['c']         # ❌ KeyError!  (없는 키로 직접 접근하면 에러)
some_dict.get('c')     # None          (에러 없이 None 반환)
some_dict.get('c', 0)  # 0             (기본값 지정 가능)
```

> `some_dict['some_key']`처럼 **대괄호로 직접 접근**하면 키가 없을 때 **에러가 발생할 수 있다.** 안전하게 조회하려면 `.get()`을 쓰는 게 정석.

### `keys()` / `values()` / `items()`

셋 다 **이터러블(iterable) 객체**를 반환한다. (리스트가 아니라 `dict_keys`, `dict_values`, `dict_items` 타입)

```python
d = {'a': 1, 'b': 2}

d.keys()      # dict_keys(['a', 'b'])
d.values()    # dict_values([1, 2])
d.items()     # dict_items([('a', 1), ('b', 2)])   ← 튜플로 묶여있는 것과 비슷

for k, v in d.items():
    print(k, v)     # a 1 / b 2
```

### `setdefault()`

키와 연결된 값을 반환한다. **키가 없다면**, `default`와 연결한 키를 딕셔너리에 **추가**하고 `default`를 반환한다.

```python
d = {'a': 1}

d.setdefault('a', 100)   # 'a'는 이미 있음 → 1 반환 (변화 없음)
d.setdefault('b', 100)   # 'b'는 없음 → d에 'b':100 추가하고 100 반환
print(d)                 # {'a': 1, 'b': 100}
```

### `update()`

**다른 딕셔너리(또는 키-값 쌍)** 로 딕셔너리를 갱신하고, **기존 키는 덮어쓴다.**

- 딕셔너리를 넣어도 되고, 키워드 인자를 넣어도 된다.

```python
d = {'a': 1, 'b': 2}

d.update({'b': 20, 'c': 3})   # b는 덮어씀, c는 새로 추가
print(d)     # {'a': 1, 'b': 20, 'c': 3}

d.update(d=4, e=5)            # 인자로도 가능
print(d)     # {'a': 1, 'b': 20, 'c': 3, 'd': 4, 'e': 5}
```

### `pop()`

키를 제거하고 연결된 값을 반환한다. 없으면 **에러**(또는 `default`가 있으면 `default`를 반환).

```python
d = {'a': 1, 'b': 2}

d.pop('a')          # 1 반환, d = {'b': 2}
d.pop('z')          # ❌ KeyError!
d.pop('z', '없음')   # '없음' 반환 (default 지정 시 에러 안 남)
```

> 리스트의 `pop()`과 달리, 딕셔너리는 **"마지막"이라는 개념이 없기 때문에** 반드시 **키를 인자로 넣어줘야 한다.** (`d.pop()`처럼 빈 채로 호출 불가)

### `clear()`

```python
d = {'a': 1, 'b': 2}
d.clear()
print(d)     # {}
```

## 5. 딕셔너리의 확장 — `defaultdict`

파이썬 내장 모듈 **`collections`**에서 제공하는 **딕셔너리의 확장판.**

- 딕셔너리에 **존재하지 않는 키를 조회할 때 자동으로 기본값을 생성**해주는 자료구조.
- **KeyError 걱정 없이** 값을 수정하거나 추가할 때 매우 유용하다.

### 기본 딕셔너리 vs `defaultdict` — 문자 개수 세기 비교

**상황:** 주어진 문자열에서 각 문자의 등장 횟수 세기

```python
# 기존: 기본 딕셔너리
text = 'banana'
counts = {}

for char in text:
    if char not in counts:      # 키가 있는지 매번 확인해야 함
        counts[char] = 0
    counts[char] += 1

print(counts)   # {'b': 1, 'a': 3, 'n': 2}
```

```python
# 개선: defaultdict 활용
from collections import defaultdict

text = 'banana'
counts = defaultdict(int)       # 기본값을 정수(0)로 설정

for char in text:
    # 키 확인 불필요! 없으면 0으로 자동 생성 후 +1
    counts[char] += 1

print(counts)   # defaultdict(<class 'int'>, {'b': 1, 'a': 3, 'n': 2})
```

> **핵심 차이:** 기본 딕셔너리는 매번 `if char not in counts:`로 **키 존재 여부를 직접 확인**해야 한다. `defaultdict(int)`는 없는 키를 조회하는 순간 **자동으로 `0`(int의 기본값)을 만들어주므로** 그 확인 코드가 필요 없어진다.

### `defaultdict` 활용 예제 — 색깔별 과일 분류하기

```python
from collections import defaultdict

fruits = [('red', 'apple'), ('yellow', 'banana'), ('red', 'cherry')]
fruit_by_color = defaultdict(list)     # 기본값을 빈 리스트로 설정

for color, fruit in fruits:
    # color 키가 없으면 빈 리스트 생성 -> append 바로 가능
    fruit_by_color[color].append(fruit)

print(fruit_by_color)
# defaultdict(<class 'list'>, {'red': ['apple', 'cherry'], 'yellow': ['banana']})
```

### 자주 쓰는 패턴 정리

| 패턴 | 초기화 | 용도 |
|------|--------|------|
| **숫자 세기** | `defaultdict(int)` | 키가 없으면 `0`으로 초기화 후 카운팅 |
| **그루핑 (리스트로 모으기)** | `defaultdict(list)` | 키가 없으면 **빈 리스트**로 초기화 후 `append` |

> **왜 이게 강력한가?** 그루핑 패턴은 알고리즘 문제에서 **그래프 인접 리스트 구성**이나 **분류(classification)** 작업에 특히 강력하다. 예를 들어 그래프의 간선 목록을 `{노드: [연결된 노드들]}` 형태로 만들 때, 일반 딕셔너리라면 매번 "이 노드가 이미 키로 있나?"를 확인하고 없으면 빈 리스트를 만드는 코드를 반복해야 한다. `defaultdict(list)`는 그 반복되는 방어 코드를 없애주기 때문에, 그래프 탐색(BFS/DFS) 같은 코드가 훨씬 짧고 명확해진다.

### ⚠️ 주의 — 조회만 해도 키가 생성된다

```python
d = defaultdict(int)
print(d['x'])       # 0  (조회했을 뿐인데)
print(d)             # defaultdict(<class 'int'>, {'x': 0})  ← 'x' 키가 생겨버림!
```

> 단순히 **값을 읽어보기만 해도** 그 키가 딕셔너리에 **실제로 추가**된다. `in` 연산자로 존재 여부만 확인하고 싶다면 `defaultdict`라도 `'x' in d`처럼 써야지, `d['x']`로 조회하면 의도치 않게 키가 생길 수 있으니 주의.

## 6. 세트 (Set)

세트는 **해시 테이블**을 사용해 데이터를 저장한다.

- 항목의 **고유성을 효율적으로 보장**한다.
- **추가·삭제·존재 여부 확인**이 크기에 상관없이 **매우 빠르다.**
- **수학적 집합 연산**을 간편하게 수행할 수 있는 것이 가장 큰 특징.

### 세트 메서드

| 메서드 | 설명 |
|--------|------|
| ⭐ `s.add(x)` | 세트 `s`에 항목 `x`를 추가. **이미 `x`가 있다면 변화 없음** |
| `s.update(iterable)` | 세트 `s`에 다른 iterable의 요소를 추가 |
| `s.clear()` | 세트 `s`의 모든 항목을 제거 |
| ⭐ `s.remove(x)` | 세트 `s`에서 항목 `x`를 제거. **항목 `x`가 없을 경우 KeyError** |
| `s.pop()` | 세트 `s`에서 **임의의** 항목을 반환하고, 해당 항목을 제거 |
| `s.discard(x)` | 세트 `s`에서 항목 `x`를 제거 (`remove`와 달리 **에러 없음**) |

### `add()` — 세트의 순서는 왜 랜덤일까?

```python
s = {1, 2, 3}
s.add(4)
print(s)    # {1, 2, 3, 4}  ← 순서가 항상 이 순서로 보장되는 건 아님
```

> 세트를 출력하면 순서가 뒤죽박죽인 것처럼 보여 찝찝할 수 있는데, 이건 세트가 **해시 테이블이라는 특수한 구조**로 데이터를 저장하기 때문이다. 값을 넣은 순서가 아니라 **해시값에 따라 내부적으로 배치**되므로, 리스트처럼 입력 순서를 유지하지 않는다.

### `update(iterable)`

세트에 **iterable을 풀어서** 넣는다. (리스트의 `extend`와 비슷한 개념)

```python
s = {1, 2}
s.update([3, 4, 5])
print(s)    # {1, 2, 3, 4, 5}
```

### `clear()`

```python
s = {1, 2, 3}
s.clear()
print(s)    # set()   ← 빈 세트는 반드시 set()으로 표시됨 (빈 {} 는 dict이므로)
```

### `remove(x)` vs `discard(x)` — 에러 처리 차이

```python
s = {1, 2, 3}

s.remove(1)      # OK, s = {2, 3}
s.remove(100)    # ❌ KeyError!  (항목이 없으면 에러)

s.discard(2)     # OK, s = {3}
s.discard(100)   # 에러 없음, 그냥 아무 일도 안 일어남
```

> **`remove()`가 없을 때 "KeyError"를 내는 이유:** 세트도 내부적으로는 **딕셔너리와 같은 해시 테이블 구조**를 사용한다. (세트는 "값만 있고 값(value)이 없는 딕셔너리"에 가까운 구조) 그래서 항목이 없을 때 나는 에러 이름도 `ValueError`가 아니라 딕셔너리와 똑같은 **`KeyError`**다.

### `pop()` — "임의"의 요소

```python
s = {1, 2, 3}
x = s.pop()      # 어떤 값이 나올지 예측 불가 (내부 해시 순서에 따라 결정됨)
```

> 리스트의 `pop()`은 "마지막 요소"라는 명확한 규칙이 있지만, 세트는 **순서 개념이 없기 때문에** 정말로 **임의(정해지지 않은)의 요소**가 제거·반환된다.

### ⚠️ `remove()` vs `discard()` — 어느 게 더 중요한가?

> **"조용한 실패(silent failure)" 문제 때문에 `discard()`보다 `remove()`가 더 중요하다.**

`discard()`는 항목이 없어도 에러 없이 조용히 넘어간다. 이건 편리해 보이지만, 만약 **"이 항목이 세트에 반드시 있어야 하는데 실수로 없는" 버그 상황**이라면, `discard()`는 그 사실을 **알려주지 않고 조용히 지나쳐버린다.** 반면 `remove()`는 항목이 없을 때 즉시 `KeyError`를 내서 **문제를 바로 드러낸다.** 그래서 "있는 게 확실할 때"는 `remove()`를 써서 실수를 조기에 잡아내는 게 더 안전한 습관이다.

## 7. 세트의 집합 메서드 — 수학적 집합 연산

| 메서드 | 설명 | 연산자 |
|--------|------|:------:|
| `set1.difference(set2)` | set1에는 들어있지만 set2에는 **없는** 항목으로 세트 생성 후 반환 (**차집합**) | `set1 - set2` |
| `set1.intersection(set2)` | set1과 set2 **모두** 들어있는 항목으로 세트 생성 후 반환 (**교집합**) | `set1 & set2` |
| `set1.issubset(set2)` | set1의 항목이 **모두 set2에 들어있으면** True 반환 (**부분집합**) | `set1 <= set2` |
| `set1.issuperset(set2)` | set1이 set2의 항목을 **모두 포함**하면 True 반환 (**상위집합**) | `set1 >= set2` |
| `set1.union(set2)` | set1 또는 set2에(혹은 둘 다) 들어있는 항목으로 세트 생성 후 반환 (**합집합**) | `set1 \| set2` |

```python
a = {1, 2, 3, 4}
b = {3, 4, 5, 6}

a.difference(b)      # {1, 2}       또는 a - b
a.intersection(b)     # {3, 4}       또는 a & b
a.union(b)             # {1,2,3,4,5,6}  또는 a | b

{1, 2}.issubset(a)          # True   ({1,2}는 a에 다 포함됨) 또는 {1,2} <= a
a.issuperset({1, 2})        # True   (a가 {1,2}를 다 포함함) 또는 a >= {1,2}
```

> **메서드로도, 연산자로도 쓸 수 있다.** 가독성 면에서는 연산자(`-`, `&`, `|`, `<=`, `>=`)가 더 간결하고 수학적 집합 표기와 직관적으로 일치해서 실무에서 자주 쓰인다.
