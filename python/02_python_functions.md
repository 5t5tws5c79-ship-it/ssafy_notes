# Python 함수 정리 (Functions)

## 1. 함수의 기초

### 1.1 함수의 목적

- 반복되는 코드를 **하나로 묶어 재사용**한다.
- 코드의 수정·관리를 훨씬 쉽게 만든다.

**함수란?** 특정 작업을 수행하기 위한 **재사용 가능한 코드 묶음**.
한 번 정의해두면 **함수의 이름만 불러** 사용할 수 있어, 재사용성이 높아지고 **가독성·유지보수성**이 향상된다.

#### 함수 사용 전 vs 후

```python
# 함수 사용 전 — 같은 코드를 매번 반복
total1 = 1 + 2 + 3
total2 = 10 + 20 + 30

# 함수 사용 후 — 한 번 정의하고 이름만 호출
def get_sum(a, b, c):
    return a + b + c

total1 = get_sum(1, 2, 3)       # 6
total2 = get_sum(10, 20, 30)    # 60
```

### 1.2 함수의 구조

```python
def get_sum(a, b):          # def 키워드 + 함수 이름 + 매개변수
    """두 수의 합을 반환한다."""   # 독스트링 (선택)
    result = a + b          # 함수 바디 (들여쓰기된 코드 블록)
    return result           # 반환값
```

| 구성 요소 | 설명 |
|-----------|------|
| `def` | 함수를 정의하는 키워드. 뒤에 **함수 이름**을 쓴다. |
| 매개변수 (parameter) | 함수가 받을 입력값의 이름 |
| `:` (콜론) | 콜론 다음 줄부터 **들여쓰기된 코드 블록**이 함수 바디 |
| 독스트링 (docstring) | 함수 바디 앞에 **선택적으로** 쓰는 함수 설명 (`"""..."""`) |
| 함수 바디 (body) | 실제 동작을 수행하는 코드 |
| `return` | 결과를 호출부로 반환 |

### 1.3 반환값 (Return Value)

- 함수는 **필요한 경우** 결과를 반환할 수 있다. `return` 키워드 뒤에 반환값을 명시한다.
- `return`문은 **함수의 실행을 종료**하고, 결과를 **호출부에 반환**한다.
- `return`문이 없으면? 파이썬이 알아서 **`None`을 반환**한다.

```python
def add(a, b):
    return a + b

def greet(name):
    print(f'안녕, {name}')   # return 없음 → None 반환

result = greet('철수')
print(result)   # None
```

> **참고:** `print()` 함수는 화면에 출력만 할 뿐 **반환값이 없다**(None). "출력하는 것"과 "반환하는 것"은 다르다.

### 1.4 독스트링 (Docstring)

함수의 동작을 설명하는 문자열. 함수 바디 맨 앞에 `"""..."""`로 작성한다.

- **스타일 가이드:** 여는 `"""` **바로 뒤에서** 설명을 시작하라고 되어 있다.
- IDE에서 함수를 사용할 때 **팝업으로 독스트링이 표시**되어, 함수 설명을 바로 볼 수 있다.

```python
def get_sum(a, b):
    """두 정수를 받아 합을 반환하는 함수."""
    return a + b
```

## 2. 인자 (Arguments)

### 2.1 매개변수(parameter)와 인자(argument)

| 시점 | 용어 |
|------|------|
| 함수를 **정의**할 때 받는 이름 | **매개변수 (parameter)** |
| 함수를 **호출**할 때 넘기는 실제 값 | **인자 (argument)** |

```python
def add(a, b):        # a, b → 매개변수
    return a + b

add(3, 5)             # 3, 5 → 인자
```

#### 매개변수 이름은 아무렇게나 지어도 되나요?

- **문법상**으로는 변수 이름 규칙만 지키면 `a`, `b`, `data` 등 자유롭게 써도 된다.
- **권장:** 정의할 때 쓴 이름을 함수 내부에서 **똑같이** 사용하고, 가독성을 위해 `x`, `y`보다 `name`, `age`처럼 **어떤 데이터인지 알 수 있는 명확한 이름**을 쓰자.

### 2.2 다양한 인자의 종류

#### 위치 인자 (Positional Argument)

함수 호출 시 **인자의 위치(순서)** 에 따라 전달되는 인자. **반드시 값을 전달**해야 한다.

```python
def sub(a, b):
    return a - b

sub(10, 3)    # a=10, b=3 → 7  (위치 순서대로 전달)
```

#### 기본 인자 값 (Default Argument Value)

위치 인자를 개선한 형태. 함수 **정의**에서 매개변수에 **기본값을 할당**한다. 호출 시 인자를 전달하지 않으면 **기본값이 사용**된다.

```python
def greet(name, msg='안녕하세요'):
    print(f'{name}님, {msg}')

greet('철수')              # 철수님, 안녕하세요   (msg 기본값 사용)
greet('영희', '반가워요')   # 영희님, 반가워요     (기본값 덮어씀)
```

#### 키워드 인자 (Keyword Argument)

함수 호출 시 **인자의 이름과 함께** 전달하는 인자.

- 매개변수와 인자를 순서로 일치시키지 않고, **이름을 명시**해 특정 매개변수에 값을 할당한다.
- 인자의 **순서는 중요하지 않다.**
- ⚠️ 단, 호출 시 **키워드 인자는 위치 인자보다 뒤에** 와야 한다.

```python
def introduce(name, age):
    print(f'{name}, {age}살')

introduce(age=25, name='철수')   # 순서 무시 OK → 철수, 25살

# 위치 + 키워드 섞어 쓸 때
introduce('철수', age=25)        # OK
introduce(name='철수', 25)       # ❌ SyntaxError! (키워드 뒤에 위치 인자 불가)
```

#### 왜 위치 인자가 앞에 와야 할까? — 순서의 모호성

- **위치 인자**는 "첫 번째, 두 번째"라는 **순서에 의존**해 값이 전달된다.
- **키워드 인자**는 순서를 무시하고 이름으로 직접 지정한다.
- 이미 키워드 인자로 **순서가 깨진 상태**에서 다시 위치 인자를 던지면, 그 값이 **몇 번째 자리**에 들어가야 할지 판단할 수 없게 된다. (매개변수가 3개 이상일 때 특히 모호해진다.)

### 2.3 가변 인자 (`*args`, `**kwargs`)

#### `*args` — 임의의 인자 목록

- 정해지지 않은 **개수의 인자**를 처리한다.
- 여러 개의 인자를 **튜플(tuple)** 로 묶어서 받는다.

```python
def my_sum(*args):
    print(args)          # 튜플로 들어옴
    return sum(args)

my_sum(1, 2, 3)          # args = (1, 2, 3) → 6
my_sum(1, 2, 3, 4, 5)    # args = (1, 2, 3, 4, 5) → 15
```

> **💡 궁금증 — `*args`에 타입이 안 박혀 있으니 아무거나 다 넣을 수 있는 것 아닌가? 에러 나기 딱 좋은 상황 아닌가?**
> 맞다. `*args`는 타입을 강제하지 않으므로 `my_sum(1, 'a', [3])`처럼 아무거나 넣을 수 있고, 그러면 안쪽 `sum()`에서 **TypeError**가 난다.
> 그래서 실무에선 필요할 때 함수 내부에서 **타입별로 분기(검사)** 해서 처리하는 게 일반적이다. 예:
> ```python
> def process(*args):
>     for x in args:
>         if isinstance(x, (int, float)):
>             ...   # 숫자 처리
>         elif isinstance(x, str):
>             ...   # 문자열 처리
> ```
> 즉, "아무거나 받을 수 있다"는 유연함의 대가로 **타입 안정성은 개발자가 직접 챙겨야** 한다.

#### `**kwargs` — 임의의 키워드 인자 목록

- 여러 개의 **키워드 인자**를 **딕셔너리(dict)** 로 묶어서 받는다.
- 입력이 딕셔너리로 들어온다는 걸 인지하고 처리한다.

```python
def my_func(**kwargs):
    print(kwargs)        # 딕셔너리로 들어옴

my_func(a=1, b=2)        # {'a': 1, 'b': 2}
```

### 2.4 인자 5종의 권장 순서

다섯 종류를 모두 쓰고 싶다면 이 순서를 권장한다:

> **위치 인자 → 기본 인자 → 가변 인자(`*args`) → 가변 키워드 인자(`**kwargs`)**

절대적인 규칙은 아니고 상황에 맞게 유연하게 조정할 수 있지만, 이 순서가 파이썬 문법상 자연스럽고 모호함이 없다.

#### 모든 인자를 적용한 예시

```python
def func(pos1, pos2, default_arg='default', *args, **kwargs):
    print('pos1:', pos1)
    print('pos2:', pos2)
    print('default_arg:', default_arg)
    print('args:', args)
    print('kwargs:', kwargs)

func(1, 2, 3, 4, 5, 6, key1='value1', key2='value2')
```

**출력:**

```
pos1: 1
pos2: 2
default_arg: 3
args: (4, 5, 6)
kwargs: {'key1': 'value1', 'key2': 'value2'}
```

- `1`, `2` → 위치 인자 `pos1`, `pos2`
- `3` → 기본 인자 `default_arg`를 덮어씀
- `4, 5, 6` → 남는 위치 인자들이 `*args` 튜플로
- `key1=..., key2=...` → 키워드 인자들이 `**kwargs` 딕셔너리로

### 2.5 (참고) `print()` 함수는 어떻게 생겼나? — Python 3.11 기준

파이썬 공식 문서에 정의된 `print()`의 시그니처:

```python
print(*objects, sep=' ', end='\n', file=sys.stdout, flush=False)
```

| 매개변수 | 역할 | 기본값 |
|----------|------|--------|
| `*objects` | 출력할 값들 (개수 제한 없음 → 가변 인자) | — |
| `sep` | 값 사이 구분자 | `' '` (공백) |
| `end` | 출력 끝에 붙는 문자 | `'\n'` (줄바꿈) |
| `file` | 출력 대상 | `sys.stdout` (화면) |
| `flush` | 버퍼 즉시 비우기 여부 | `False` |

```python
print('a', 'b', 'c')                # a b c
print('a', 'b', 'c', sep='-')       # a-b-c
print('hello', end='')              # 줄바꿈 없이 출력
print('a', 'b', sep=', ', end='!')  # a, b!
```

> 앞에서 배운 `*args`, 기본 인자, 키워드 인자가 **실제 내장 함수에 그대로 쓰이고 있음**을 확인할 수 있다.

## 3. 재귀 함수 (Recursion)

**재귀 함수란?** 함수 내부에서 **자기 자신을 호출**하는 함수.

- 재귀를 이용해 문제를 **작은 단위의 문제로 분할**하고, 분할된 결과를 조합해 최종 결과를 도출한다.
- ⚠️ 반드시 **종료 조건(base case)** 을 설정해 재귀를 멈춰야 한다.

### 3.1 팩토리얼 예시

```python
def factorial(n):
    if n <= 1:          # 종료 조건
        return 1
    return n * factorial(n - 1)   # 자기 자신을 재귀 호출

print(factorial(5))     # 120
```

### 3.2 동작 구조 — 콜 스택 (Call Stack)

1. 호출이 일어나면 아직 평가되지 않은 채 **콜 스택에 계속 쌓이기만** 한다.
   `factorial(5)` → `factorial(4)` → ... → `factorial(1)`
2. 종료 조건에 도달하면, 그때부터 **반환값들이 스택에서 빠지면서** 거꾸로 연산된다.
   `1 → 2×1 → 3×2 → 4×6 → 5×24 = 120`

### 3.3 재귀의 특징

**장점**
- 특정 알고리즘을 표현할 때 **변수 사용이 줄고**, 코드 **가독성**이 높아진다.
- 자연스러운 표현, 간결성, 수학적 문제 해결에 강하다.

**주의점**
- 종료 조건을 **명확히** 할 것.
- 반복되는 호출이 **종료 조건을 향하도록** 할 것.
- **스택 오버플로**가 날 수 있고, **메모리 사용량이 많다.**
- 복잡한 재귀는 오히려 **가독성을 떨어뜨릴** 수 있다.

## 4. 내장 함수 (Built-in Functions)

파이썬에 **기본적으로 내장된 함수**들.

> **"바퀴를 다시 발명하지 마세요 (Don't reinvent the wheel)"**
> 이미 검증되고 최적화된 기능이 있다면 활용하는 게 좋다. 내장 함수는 매우 빠르고 효율적으로 동작하도록 만들어져 있다.

### 4.1 자주 쓰는 내장 함수

```python
print()    # 출력
len()      # 길이
max()      # 최댓값
min()      # 최솟값
sum()      # 합계
sorted()   # 정렬
```

## 5. 스코프 (Scope)

### 5.1 함수와 스코프

파이썬은 함수 내부에 **로컬 스코프(local scope)** 를 생성하고, 그 외의 공간을 **글로벌 스코프(global scope)** 로 구분한다.

- **로컬 변수:** 함수 안에서 만들어져 함수 안에서만 유효
- **글로벌 변수:** 함수 밖에서 만들어져 어디서나 접근 가능

```python
x = 'global'      # 글로벌 변수

def func():
    x = 'local'   # 로컬 변수 (함수 안에서만 유효)
    print(x)      # local

func()
print(x)          # global  (바깥은 영향 없음)
```

#### 변수의 수명 주기 (Lifecycle)

스코프는 변수의 **수명 주기**와 연관된다.

| 스코프 | 수명 |
|--------|------|
| **빌트인 (Built-in)** | 파이썬 실행 내내 항상 존재 |
| **글로벌 (Global)** | 모듈이 호출된 시점부터 인터프리터가 끝날 때까지 유지 |
| **로컬 (Local)** | 함수가 **호출될 때 생성**, 함수가 **종료될 때 소멸** |

### 5.2 이름 검색 규칙 — LEGB Rule

이름은 **네임스페이스(namespace)** 에 저장되며, 파이썬은 변수를 찾을 때 **L → E → G → B** 순서로 탐색한다.

| 순서 | 스코프 | 설명 |
|:----:|--------|------|
| **L** | Local | 지역 (현재 함수 내부) |
| **E** | Enclosed | 지역 **한 단계 위**의 (감싸는) 함수 범위 |
| **G** | Global | 전역 (모듈 최상단) |
| **B** | Built-in | 파이썬 내장 |

- 함수 내에서는 바깥 스코프의 변수에 **접근은 가능**하지만 **수정은 할 수 없다.** (이 특성을 잘 활용해야 한다.)

#### LEGB 동작 예시

```python
x = 'G'
y = 'G'

def outer_func():
    x = 'E'
    y = 'E'

    def inner_func(y):
        z = 'L'
        print(y, z)   # E P L → 매개변수 y='P', z='L'

    inner_func('P')
    print(x, y)       # E E

outer_func()
print(x, y)           # G G
```

> **⚠️ 흔한 실수 — 이름 가리기(shadowing):**
> `sum`이라는 이름을 글로벌에서 변수로 써버리면, 빌트인 스코프의 내장 함수 `sum()`을 더 이상 쓸 수 없게 된다. (LEGB에서 글로벌이 빌트인보다 먼저 검색되기 때문.) 내장 함수 이름을 변수명으로 쓰지 않도록 주의하자.

### 5.3 `global` 키워드

바깥(글로벌)에 있는 변수를 함수 안에서 **수정해야만 할 때** 사용한다.

```python
num = 0

def increment():
    global num      # "바깥의 num을 수정하겠다" 선언
    num += 1

increment()
print(num)          # 1
```

**주의사항**
- `global` 선언은 **참조하기 전에** 해야 한다. (선언 전에 참조 불가)
- **매개변수**에는 `global` 키워드를 사용할 수 없다.

> `global`은 편리하지만 남용하면 코드 추적이 어려워지므로 꼭 필요할 때만 쓰자.

## 6. 함수 스타일 가이드

### 6.1 함수 이름 규칙

- **소문자와 언더스코어** 사용 (`snake_case`)
- **동사로 시작**해 함수의 동작을 설명 (`get_user`, `calculate_total`)
- 약어 사용 지양
- 자주 쓰는 접두사: `get_` / `set_` 등
- 이름만으로 무엇을 하는지 명확하게
- `True`/`False`를 반환하는 함수는 `is_`, `has_`로 시작 추천 (`is_valid`, `has_permission`)
- 프로젝트 전체에 **일관성**을 지키는 것이 가독성에 도움

### 6.2 단일 책임 원칙 (SRP, Single Responsibility Principle)

> 모든 객체(함수)는 **하나의 명확한 목적과 책임**을 가져야 한다.

| 원칙 | 내용 |
|------|------|
| 명확한 목적 | 한 가지 작업만, 이름으로 목적을 명확히 표현 |
| 책임 분리 | 데이터 검증·처리·저장을 **별도 함수**로 분리, 각 함수는 독립적으로 동작 |
| 유지보수성 | 작은 단위 함수로 나눠 관리 → 코드 수정 시 **영향 범위 최소화** |

```python
# 올바른 설계 예 — 책임을 나눔
def validate_password(pw): ...   # 유효성 검사
def hash_password(pw): ...       # 암호화 및 저장
def send_welcome_email(user): ...# 환영 이메일 발송

# main에서 순차적으로 실행
```

## 7. 패킹·언패킹과 반환

### 7.1 패킹과 언패킹 (Packing & Unpacking)

#### 패킹 (Packing)

여러 개의 데이터를 **하나의 컬렉션으로 모아 담는** 과정.

- 한 변수에 콤마로 구분한 값을 넣으면 **자동으로 튜플**로 처리된다.
- `*args`, `**kwargs`가 대표적. (`print()`도 패킹의 예시)

```python
packed = 1, 2, 3       # 자동으로 튜플 → (1, 2, 3)
```

#### 언패킹 (Unpacking)

컬렉션에 담긴 데이터를 **개별 요소로 펼쳐놓는** 과정.

```python
# 다중 할당
a, b, c, d, e = (1, 2, 3, 4, 5)   # 각 변수에 1,2,3,4,5

# 함수 호출 시 * 사용 → 개별 위치 인자로 전달
def my_function(x, y, z):
    print(x, y, z)

names = ['alice', 'jane', 'peter']
my_function(*names)    # alice jane peter
```

#### `**`를 활용한 언패킹

딕셔너리 앞에 `**`를 붙이면, **키 == 매개변수 이름**이 맞을 때 **키워드 인자**로 전달된다.

```python
def introduce(name, age):
    print(name, age)

info = {'name': '철수', 'age': 25}
introduce(**info)      # 철수 25
```

#### `*` / `**` 정리표

| 구분 | 상황 | `*` 연산자 | `**` 연산자 |
|------|------|-----------|-------------|
| **패킹** | 함수 **정의** 시 | 여러 위치 인자를 하나의 **튜플**로 받음 | 여러 키워드 인자를 하나의 **딕셔너리**로 받음 |
| **언패킹** | 함수 **호출** 시 | 리스트/튜플을 개별 **위치 인자**로 전달 | 딕셔너리를 개별 **키워드 인자**로 전달 |

### 7.2 함수와 반환 (다시 정리)

함수는 **언제나 `None`을 포함한 단 하나의 객체**만 반환할 수 있다.

- 여러 값을 반환하는 것처럼 보여도, 실제로는 **하나의 튜플로 패킹**해서 반환한다.

```python
def min_max(numbers):
    return min(numbers), max(numbers)   # 튜플로 자동 패킹

result = min_max([3, 1, 5])
print(result)          # (1, 5)  ← 튜플 하나

low, high = min_max([3, 1, 5])   # 언패킹해서 사용
print(low, high)       # 1 5
```

> **핵심 3가지:** ① 하나만 리턴 ② 여러 값은 튜플로 자동 패킹 ③ 튜플은 언패킹해서 쓸 수 있음

## 8. 람다 표현식 (Lambda)

**익명 함수**를 만드는 표현식. 한 줄로 간단히 함수를 정의한다.

```python
# 일반 함수
def addition(x, y):
    return x + y

# 람다 표현식 (같은 기능)
addition = lambda x, y: x + y
```

**언제 쓰나?**
- 쓰고 바로 버리는 **일회성 함수**가 필요할 때
- 함수의 **매개변수가 함수를 요구할 때** (예: `sorted`의 `key`)
- 간결하게, 필요한 곳에 즉석에서 로직을 작성할 수 있다.

### 8.1 `sorted` + 람다 예시

```python
# 학생 데이터 (이름, 나이) 튜플의 리스트
students = [('지민', 25), ('서준', 20), ('민우', 30)]

# 1) lambda 미사용 — 정렬 기준 함수를 따로 정의해야 함
def get_age(student):
    return student[1]

result = sorted(students, key=get_age)
# [('서준', 20), ('지민', 25), ('민우', 30)]

# 2) lambda 사용 — 즉석에서 기준 지정 (간결)
result = sorted(students, key=lambda student: student[1])
print(result)
# [('서준', 20), ('지민', 25), ('민우', 30)]
```

> **`sorted`의 `key`:** `key` 매개변수에 **함수를 전달**해 "무엇을 기준으로 정렬할지"를 지정한다. 여기서는 튜플의 인덱스 `[1]`(나이)을 기준으로 정렬. 이렇게 **정렬 기준으로 람다를 활용**하는 것이 대표적인 쓰임새다.
