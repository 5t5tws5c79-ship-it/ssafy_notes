## 2. 표현식과 값

### 2.1 표현식 (Expression)

**표현식이란?** 하나의 값으로 평가(evaluate)될 수 있는 모든 코드.

```python
3 + 5      # 표현식
x > 10     # 표현식
```

### 2.2 값 (Value)

**값이란?** 표현식이 평가된 결과. 더 이상 평가하거나 계산할 수 없는 데이터 조각.

- 종류: 숫자값, 문자열값, 불리언값 등
- `3 + 5`는 **표현식**이지만 그 자체로 값이 아니며, 평가를 거쳐야 값(`8`)이 된다.

| 구분 | 설명 | 예시 |
|------|------|------|
| 표현식 | 평가될 수 있는 코드 | `3 + 5` |
| 값 | 평가가 끝난 결과 | `8` |

## 3. 변수와 메모리

### 3.1 변수 (Variable)

**변수란?** 값을 다시 사용하기 위해 저장해놓은 것.

#### 변수 할당 (Assignment)

표현식이 만들어낸 값에 이름을 붙이는 과정. 이를 **할당문**이라 한다.

```python
degrees = 36.5
```

- 값 `36.5`를 변수 `degrees`에 할당했다.
- `=` 는 **할당 연산자**.

#### 변수명 규칙

- 영문 알파벳, 언더스코어(`_`), 숫자로 구성
- 숫자로 시작할 수 없음
- 대소문자 구분 (`age` ≠ `Age`)
- 예약어(keyword)는 변수명으로 사용 불가

### 3.2 메모리와 객체 (Object)

#### 메모리 주소

값은 메모리 어딘가에 존재하며, 그 위치를 **메모리 주소**라 한다.
사물함 번호처럼 기계적으로 매겨진 숫자 주소.

#### 객체 (Object)

**값 + 타입 + 주소 정보를 묶은 것**을 객체라고 부른다.
모든 객체는 고유한 **id / 타입 / 값**을 가진다.

#### 변수 = 객체를 가리키는 이름표

변수는 특정 객체를 가리키는 **이름표(reference / point to)** 다.

> **비유 (주소록):**
> 주소록의 '내 친구 철수'라는 항목 자체가 실제 집 주소는 아니다.
> 하지만 그 이름을 보고 실제 집을 찾아갈 수 있다.
> `my_friend = '김철수'` 라고 쓰면, 변수 `my_friend`는 `'김철수'`라는 객체를 가리키는 이름표 역할을 한다.
> 파이썬은 이 이름표를 보고 실제 객체가 저장된 메모리 주소를 찾아 작업을 수행한다.

| 용어 | 핵심 정의 | 비유 (주소록) |
|------|-----------|---------------|
| 객체 (Object) | 데이터(값, 타입, 행동)의 실체 | '김철수'라는 실제 사람 |
| 메모리 주소 | 객체가 저장된 고유한 위치 | 김철수의 실제 집 주소 |
| 변수 (Variable) | 객체를 가리키는 이름표 | 주소록에 저장된 '내 친구 철수'라는 이름 |

### 3.3 할당문 동작 순서

`variable = expression` 이 실행될 때:

1. **오른쪽 표현식 평가**
   할당 연산자(`=`)의 오른쪽 표현식 전체를 계산해 하나의 결과값(객체)을 만든다.

2. **왼쪽 변수명 확인**
   - 처음 사용된 이름이면 → 새로운 '이름표'를 준비
   - 이미 존재하는 이름이면 → 기존 '이름표'를 그대로 사용

3. **변수명과 결과값 연결 (참조)**
   왼쪽 변수명이 오른쪽에서 만들어진 결과값을 가리키도록 연결한다.
   기존에 다른 객체를 가리키고 있었다면 그 연결은 끊어지고, 새 객체와의 연결만 남는다.

### 3.4 재할당 (Reassignment)

변수를 새로운 값으로 변경하는 것.

```python
degrees = 36.5
degrees = 40.0   # 재할당 → 이제 40.0을 가리킴
```

## 4. 타입 (Type)

**데이터 타입이란?** 값의 종류와, 그 값으로 할 수 있는 동작을 정하는 지표.

### 4.1 타입이 필요한 이유

- 숫자는 더하기(`+` → 덧셈), 문자열은 결합(`+` → 이어붙이기)
- 숫자와 문자는 서로 더할 수 없음

```python
3 + 5          # 8       (숫자 덧셈)
"a" + "b"      # "ab"    (문자열 결합)
3 + "a"        # TypeError!
```

### 4.2 데이터 타입 종류 (5가지 분류)

| 분류 | 타입 |
|------|------|
| Numeric (숫자) | `int`, `float`, `complex` |
| Text (문자) | `str` |
| Sequence (순서 O) | `list`, `tuple`, `range` |
| Non-sequence (순서 X) | `set`, `dict` |
| 기타 | `bool`, `None`, `function` 등 |

> **참고:** 파이썬에는 C 언어 같은 `char` 타입이 **따로 없다.** 문자 하나도 길이 1짜리 `str`로 취급한다.

## 5. 숫자형 (Numeric Types)

### 5.1 정수형 (Integer, int)

정수를 표현하는 자료형. 소수점 없는 숫자.

```python
age = 25
count = -3
big = 1_000_000   # 언더스코어로 자릿수 구분 가능
```

### 5.2 실수형 (Float)

소수점이 있는 숫자를 표현하는 자료형.

```python
pi = 3.14
temp = -0.5
```

#### 지수 표현 (Exponential)

매우 크거나 작은 수를 `e`(또는 `E`)를 써서 표현한다. `e` 뒤의 숫자는 10의 거듭제곱.

```python
a = 3e2      # 3 × 10^2  = 300.0
b = 1.5e-3   # 1.5 × 10^-3 = 0.0015
```

### 5.3 산술 연산자 (Arithmetic Operators)

| 연산자 | 의미 | 예시 | 결과 |
|--------|------|------|------|
| `+` | 덧셈 | `5 + 2` | `7` |
| `-` | 뺄셈 | `5 - 2` | `3` |
| `*` | 곱셈 | `5 * 2` | `10` |
| `/` | 나눗셈 (항상 float) | `5 / 2` | `2.5` |
| `//` | 몫 (정수 나눗셈) | `5 // 2` | `2` |
| `%` | 나머지 (modulo) | `5 % 2` | `1` |
| `**` | 거듭제곱 | `5 ** 2` | `25` |

#### 연산자 우선순위 (Operator Precedence)

파이썬도 수학처럼 정해진 순서에 따라 연산을 수행한다.

```python
-2 ** 4    # 결과: -16
```

- `**`(거듭제곱)가 단항 `-`(부호)보다 **먼저** 계산되기 때문에 `-(2 ** 4)` = `-16`이 된다.
- 헷갈리기 쉬우므로 **괄호로 명확하게 표현**하는 것이 중요하다.

```python
(-2) ** 4  # 결과: 16
```

## 6. 시퀀스 타입 (Sequence Type)

**순서(order)가 있는** 자료형. (여기서 '순서'는 값이 들어간 차례가 있다는 뜻이지, **정렬(sorted)** 되어 있다는 뜻이 아니다.)

### 6.1 시퀀스의 공통 기능

- **인덱싱 (Indexing)** — 특정 위치의 값 꺼내기
- **슬라이싱 (Slicing)** — 일정 범위 잘라내기
- **길이 (Length)** — `len()`으로 개수 확인
- **반복 (Iteration)** — 반복문으로 각 값을 하나씩 순서대로 꺼내 사용

```python
my_data = 'hello'   # 문자열도 사실 시퀀스다!

my_data[0]          # 'h'  (인덱싱)
my_data[1:3]        # 'el' (슬라이싱)
len(my_data)        # 5    (길이)

for char in my_data:  # 반복
    print(char)
```

### 6.2 문자열 (str)

**문자들의 순서가 있는, 변경 불가능한(immutable) 시퀀스** 자료형.

#### 문자열 만들기 — 따옴표 활용

```python
a = 'hello'
b = "hello"
```

- 작은따옴표(`'`)와 큰따옴표(`"`)는 기능상 같다.
- 한 프로젝트 안에서 **혼용하지 않는 것을 권장**한다.

#### 문자열 안에 따옴표 넣기

서로 **다른 종류**의 따옴표를 사용한다.

```python
msg = "It's a book"      # 작은따옴표를 문자열 안에
quote = 'He said "hi"'   # 큰따옴표를 문자열 안에
```

#### 이스케이프 시퀀스 (Escape Sequence)

백슬래시(`\`)로 특수 문자를 표현한다.

| 이스케이프 | 의미 |
|-----------|------|
| `\n` | 줄바꿈 (new line) |
| `\t` | 탭 |
| `\\` | 백슬래시 문자 그대로 |
| `\'` | 작은따옴표 |
| `\"` | 큰따옴표 |

#### 여러 줄 문자열

삼중 따옴표(`"""` 또는 `'''`)를 사용한다.

```python
text = """여러 줄
문자열을
쓸 수 있다"""
```

#### f-string (formatted string)

문자열 앞에 `f`를 붙이고 `{}` 안에 변수나 표현식을 넣으면, 그 값이 문자열에 삽입된다.

```python
name = 'Alice'
age = 25

print(f'제 이름은 {name}이고 {age}살입니다.')
# 제 이름은 Alice이고 25살입니다.

print(f'내년엔 {age + 1}살')   # {} 안에 표현식도 가능 → 내년엔 26살
```

### 6.3 인덱스 (Index)

#### 왜 인덱스는 0부터 시작할까?

**거리(offset)의 개념**으로 이해하면 쉽다. 인덱스는 "시작점에서 얼마나 떨어져 있는가"를 뜻한다. 맨 앞 요소는 시작점에서 **0만큼** 떨어져 있으므로 인덱스가 `0`이다.

#### `'hello'`의 인덱스 표

파이썬은 뒤에서부터 세는 **음수 인덱스**도 지원한다.

| 문자 | `h` | `e` | `l` | `l` | `o` |
|------|-----|-----|-----|-----|-----|
| 양수 인덱스 | 0 | 1 | 2 | 3 | 4 |
| 음수 인덱스 | -5 | -4 | -3 | -2 | -1 |

```python
my_str = 'hello'
my_str[0]    # 'h'
my_str[-1]   # 'o'  (맨 뒤)
```

### 6.4 슬라이싱 (Slicing)

일정 범위를 잘라내는 기능. `start`는 포함, `stop`은 **미포함**.

```python
my_sequence[start:stop:step]
```

세 값 모두 상황에 따라 생략할 수 있다.

```python
my_str = 'hello'

my_str[2:4]    # 'll'   (인덱스 2 ~ 3)
my_str[:3]     # 'hel'  (처음 ~ 인덱스 2)
my_str[::2]    # 'hlo'  (처음부터 끝까지 2칸씩)
my_str[::-1]   # 'olleh' (거꾸로! 파이썬의 강력한 기능)
```

| 구성 요소 | 의미 | 생략 시 기본값 |
|-----------|------|----------------|
| `start` | 시작 인덱스 (포함) | 처음 (0) |
| `stop` | 끝 인덱스 (미포함) | 끝까지 |
| `step` | 건너뛰는 간격 | 1 |

### 6.5 문자열은 변경 불가능 (Immutable)

문자열은 **변경 불가능한 시퀀스**라서, 일부 문자만 바꾸려 하면 에러가 난다.

```python
my_str = 'hello'
my_str[1] = 'a'   # ❌ TypeError!
```

문자열을 "바꾸려면" 새로운 문자열을 **새로 만들어** 재할당해야 한다. (기존 객체를 수정하는 것이 아니라, 완전히 새 객체를 만드는 것 — 재할당과 헷갈리면 안 된다.)

```python
my_str = 'hello'
my_str = 'hallo'          # 새 문자열 객체로 재할당
# 또는
my_str = 'h' + 'a' + my_str[2:]   # 'hallo'
```

> **핵심:** `my_str[1] = 'a'`(내부 수정)는 불가능하지만, `my_str = 새로운문자열`(재할당)은 가능하다. 전자는 원본을 바꾸려는 시도(불가), 후자는 이름표를 새 객체로 옮기는 것(가능)이라는 점이 다르다.

## 7. 숫자형 보충

시퀀스 타입까지 배운 뒤, 처음의 숫자형으로 돌아와 좀 더 깊이 있는 내용을 다룬다.

### 7.1 (참고) 정수형의 진법 표현

#### 코드에서 진법 표현하기 — 접두사(prefix)

| 진법 | 접두사 | 예시 | 10진수 값 |
|------|--------|------|-----------|
| 2진법 (binary) | `0b` | `0b1010` | 10 |
| 8진법 (octal) | `0o` | `0o12` | 10 |
| 16진법 (hexadecimal) | `0x` | `0xA` | 10 |

```python
a = 0b1010   # 2진법
b = 0o12     # 8진법
c = 0xA      # 16진법

print(a, b, c)   # 10 10 10
```

> **핵심:** 어떤 진법으로 작성하든 파이썬 내부에서는 **모두 같은 값**으로 저장된다.
> `print()`는 기본적으로 **10진수로 변환**해서 보여준다.

### 7.2 실수의 함정 — 부동소수점 오차 (Floating-point Error)

#### 이상한 계산 결과

```python
print(0.1 + 0.2)      # 0.30000000000000004  😵
print(0.1 + 0.2 == 0.3)  # False
```

#### 발생 원인

- 컴퓨터는 내부적으로 **2진법**을 사용한다.
- 10진수 소수 중 일부는 2진법으로 바꾸면 **무한소수**가 된다.
- 무한히 저장할 수 없으므로 **근사값**으로 저장 → 미세한 오차 발생.

#### 해결책 — `decimal` 모듈

정확한 소수 연산이 필요하면 `decimal` 모듈을 사용해 부동소수점 연산의 정확성을 보장한다.

```python
from decimal import Decimal

print(Decimal('0.1') + Decimal('0.2'))   # 0.3
```

## 8. 문장 (Statement)

**문장이란?** 파이썬이 수행하는 하나의 완결된 '명령/지시'.
할당문, 조건문, 반복문 등이 모두 문장이다.

### 8.1 표현식 vs 문장

핵심 질문: **"이 코드를 실행하면 하나의 값이 남나요?"**

| 구분 | 값이 남나요? | 설명 | 예시 |
|------|:---:|------|------|
| 표현식 (Expression) | ✅ 네 | 계산·평가되어 하나의 값으로 귀결됨 | `3 + 5`, `x > 10` |
| 문장 (Statement) | ❌ 아니요 | 어떤 동작을 하라는 '지시' | `name = '홍길동'` |

```python
3 + 5              # 표현식 → 8 이라는 값이 남음
name = '홍길동'     # 문장 → 값을 변수에 할당하라는 지시 (값이 남지 않음)
```

### 8.2 둘의 관계 — 문장은 표현식을 재료로 사용한다

- **표현식 = 재료** (예: `3 + 5`)
- **문장 = 요리법** (예: `total = 3 + 5` — 재료로 만든 값을 `total`에 담으라는 지시)

문장은 표현식을 재료로 삼아 어떤 작업을 수행한다.

## 9. 스타일 가이드와 관례

### 9.1 PEP 8

**PEP 8이란?** 코드의 **일관성과 가독성**을 향상시키기 위한 규칙·권장 사항의 모음.

#### 주요 규칙

- 들여쓰기(공백)는 **스페이스 4칸**
- 한 줄의 길이는 **79자** 이하
- 탭 사이즈를 4칸으로 설정 (안 되어 있으면 VS Code 설정을 조정)
- 변수·함수 이름은 소문자와 언더스코어(`_`) 사용 (예: `my_name`, `get_value`)
- 함수 정의, 클래스 정의 등 블록 사이에는 **빈 줄**을 추가

> 이 외에도 훨씬 많은 규칙이 있다. 필요할 때 [PEP 8 공식 문서](https://peps.python.org/pep-0008/)를 참고하자.

### 9.2 주석 (Comment)

코드에 대한 설명. 실행되지 않는다.

```python
# 한 줄 주석 (VS Code에서 Ctrl + / 로 토글 가능)

"""
여러 줄 주석
(사실은 문자열이지만 주석처럼 활용)
"""
```

> **팁:** `#` 뒤에는 **공백 한 칸**을 두는 것이 PEP 8 권장 스타일이다. (`#주석` ❌ → `# 주석` ✅)

## 11. 리스트 (List)

**리스트란?** 여러 개의 값을 **순서대로 저장**하는 **변경 가능한(mutable) 시퀀스**.

- 대괄호 `[]`로 표현하고, 각 값은 쉼표(`,`)로 구분한다.
- 숫자, 문자열, 심지어 **다른 리스트까지** 모든 종류의 데이터를 담을 수 있다.
- 값의 추가·수정·삭제 등 자유롭게 변경 가능하다.

### 11.1 리스트의 특징을 잘 보여주는 예시

```python
# 1) 여러 타입을 한꺼번에 담을 수 있다
mixed = [1, 'hello', 3.14, True]

# 2) 리스트 안에 리스트도 넣을 수 있다
nested = [1, [2, 3], 4]

# 3) 만든 뒤에도 자유롭게 바꿀 수 있다
scores = [90, 80, 70]
scores.append(100)   # 추가 → [90, 80, 70, 100]
scores[0] = 95       # 수정 → [95, 80, 70, 100]
```

#### 배열(array)이랑 뭐가 달라요?

- 다른 언어의 **배열(array)** 과 비슷한 개념인데, 파이썬에서는 이걸 **리스트(list)** 라고 부른다.
- (참고: 파이썬에도 `array` 모듈이 따로 있지만, 일반적으로는 리스트를 쓴다.)

### 11.2 리스트의 시퀀스 특징

리스트는 시퀀스이므로 **인덱싱, 슬라이싱, 길이 확인, 반복** 등 공통 기능을 모두 사용할 수 있다.

```python
my_list = [10, 20, 30, 40]

my_list[0]      # 10      (인덱싱)
my_list[1:3]    # [20, 30] (슬라이싱)
len(my_list)    # 4       (길이 — len()으로 확인)

for item in my_list:   # 반복
    print(item)
```

### 11.3 중첩 리스트 (Nested List)

리스트 안에 리스트가 들어있는 구조. 인덱스를 **연달아** 사용해 안쪽 값에 접근한다.

```python
data = [1, 2, 3, [4, 5, 6]]

data[3]        # [4, 5, 6]  → 안쪽 리스트 전체
data[3][-1]    # 6          → 안쪽 리스트의 마지막 값
data[3][0]     # 4
```

### 11.4 리스트의 가변성 (Mutable)

#### mutable ↔ immutable

| 구분 | 의미 | 예시 타입 |
|------|------|-----------|
| **mutable** (가변) | 생성 후 자유롭게 **수정·추가·삭제** 가능 | `list`, `dict`, `set` |
| **immutable** (불변) | 생성 후 **절대 변경 불가** | `str`, `tuple`, `range`, `int` |

리스트는 **mutable**이라 값을 자유롭게 바꿀 수 있다.

인덱싱으로 값 수정하기:

```python
my_list = [1, 2, 3]
my_list[0] = 100     # [100, 2, 3]
```

슬라이싱으로 여러 값 한 번에 바꾸기:

```python
my_list = [1, 2, 3, 4, 5]
my_list[1:3] = [20, 30]   # [1, 20, 30, 4, 5]
```

> **비교:** 문자열은 immutable이라 `my_str[1] = 'a'`가 **불가능**했지만, 리스트는 mutable이라 `my_list[1] = 20`이 **가능**하다.

## 12. 튜플 (Tuple)

**튜플이란?** 여러 개의 값을 **순서대로 저장**하는 **변경 불가능한(immutable) 시퀀스**.

- 소괄호 `()`로 만들고, 값은 쉼표(`,`)로 구분한다.
- 한 번 만들어지면 **절대 수정할 수 없다.**
- 소괄호는 **생략할 수도** 있다.

```python
t1 = (1, 2, 3)
t2 = 1, 2, 3      # 소괄호 생략 가능 → 같은 튜플
```

### 12.1 ⚠️ 요소가 하나인 튜플 — 후행 쉼표 필수

단일 요소 튜플은 반드시 **뒤에 쉼표**를 붙여야 한다. 안 붙이면 그냥 괄호로 감싼 값이 되어 버린다.

```python
a = (1,)     # 튜플  → type: tuple
b = (1)      # 정수  → type: int  (그냥 괄호일 뿐)
c = ('a')    # 문자열 → type: str
```

### 12.2 튜플의 시퀀스 기능

인덱싱·슬라이싱·길이 확인은 가능하다.

```python
t = (10, 20, 30)
t[0]      # 10
t[1:]     # (20, 30)
len(t)    # 3
```

### 12.3 튜플의 불변성

한 번 생성된 튜플은 내용을 **수정·추가·삭제할 수 없다.** 시도하면 **TypeError** 발생.

```python
t = (1, 2, 3)
t[0] = 100    # ❌ TypeError! (인덱스로 접근한 수정도 불가)
```

### 12.4 튜플은 언제 쓰나요?

- 개발자가 직접 쓰기보다는 **내부 동작**이나 **안전한 데이터 전달**에 주로 쓰인다.
- **다중 할당 / 값 교환(swap)** 에 활용된다.

```python
x, y = 1, 2       # 다중 할당 (사실 튜플이 쓰인 것)
x, y = y, x       # 값 교환 → x=2, y=1
```

## 13. 레인지 (Range)

**레인지란?** 연속된 정수 시퀀스를 생성하는 **변경 불가능한(immutable)** 자료형.

- 모든 숫자를 메모리에 저장하지 않고 **시작값·끝값·간격이라는 규칙만 기억** → **메모리 효율이 높다.**
- 주로 **for문**이나 함수에서 사용한다.

### 13.1 구문

```python
range(stop)
range(start, stop)
range(start, stop, step)
```

- 여기서 `start`, `stop`, `step`을 **매개변수(parameter)** 라 하고, 실제로 넣는 값을 **인자(argument)** 라 한다.

#### 매개변수 개수에 따른 동작

| 형태 | 인식 | 자동 설정 |
|------|------|-----------|
| `range(5)` | `stop`만 지정 | `start=0`, `step=1` |
| `range(2, 5)` | `start`, `stop` 지정 | `step=1` |
| `range(2, 10, 2)` | 모두 직접 지정 | 없음 |

```python
range(5)         # 0 1 2 3 4
range(2, 5)      # 2 3 4
range(2, 10, 2)  # 2 4 6 8
range(5, 0, -1)  # 5 4 3 2 1  (step 음수 → 감소)
```

### 13.2 레인지는 왜 값이 안 보일까? (Lazy Evaluation)

레인지는 **지연 평가(lazy evaluation)** 를 하므로, 필요할 때까지 실제 값을 만들지 않고 **평가 전 객체**로 둔다. 그래서 타입 캐스팅하지 않으면 내부 값이 보이지 않는다.

```python
print(range(5))         # range(0, 5)   ← 값이 안 보임
print(list(range(5)))   # [0, 1, 2, 3, 4]  ← list로 변환해야 보임
```

### 13.3 레인지 규칙 정리

- `step` 기본값은 `1`.
- `start`부터 `stop`을 **향해 증가**한다.
- `step`이 음수일 때는 `start`부터 `stop`을 **향해 감소**하며, 이때 `start`가 `stop`보다 커야 한다. 아니면 **빈 결과(`[]`)** 가 나온다.

## 14. 딕셔너리 (Dictionary)

**딕셔너리란?** **키-값(key-value) 쌍**으로 이루어진, **순서와 중복이 없는** 변경 가능한(mutable) 자료형.

- 중괄호 `{}` 안에 키-값 쌍을 쉼표(`,`)로 구분한다.
- 키와 값은 콜론(`:`)으로 구분한다.

```python
person = {'name': '홍길동', 'age': 25}
```

### 14.1 키(key)와 값(value)

| 구분 | 역할 | 규칙 |
|------|------|------|
| **키 (key)** | 식별용 고유 이름표 | **중복 불가** |
| **값 (value)** | 키에 해당하는 데이터 | 값끼리 **중복 가능** |

### 14.2 순서와 인덱스

- 순서가 없으므로 **인덱스도 없다.** (`d[0]` 같은 접근 ❌ — 이건 키 조회임)
- 단, **파이썬 3.7 이상**부터는 개발 편의를 위해 **입력한 순서가 출력 시 그대로 유지**된다.
- 그래도 딕셔너리의 핵심은 **"순서가 없는 자료형"** 이라는 점과 **키를 통한 접근**이라는 점이다.

### 14.3 키로 쓸 수 있는 타입

키는 고유해야 하므로 **immutable 자료형만** 사용 가능하다.

| 키로 사용 | 가능 여부 |
|-----------|-----------|
| `str`, `int`, `float`, `tuple` | ✅ 가능 |
| `list`, `dict` | ❌ 불가능 (mutable이라서) |

값(value)에는 아무 타입이나 자유롭게 넣을 수 있다.

### 14.4 값 접근·추가·변경

```python
person = {'name': '홍길동', 'age': 25}

# 접근 — 키로 값 꺼내기 (특정 값 하나 찾는 건 리스트보다 빠름)
person['name']     # '홍길동'

# 추가
person['city'] = 'Seoul'   # {'name': '홍길동', 'age': 25, 'city': 'Seoul'}

# 변경
person['age'] = 26         # age가 26으로 수정됨
```

### 14.5 딕셔너리는 언제 쓰나요?

- 순서가 필요 없고, 데이터에 **의미 있는 이름(키)** 을 붙여 관리하고 싶을 때.
- **JSON 파싱** 같은 작업에서 자주 쓰인다.

## 15. 세트 (Set)

**세트란?** **순서와 중복이 없는** 변경 가능한(mutable) 자료형.

- 중괄호 `{}`로 만든다.
- **주의:** 빈 중괄호 `{}`는 **딕셔너리**가 된다. 빈 세트는 반드시 `set()`으로 만든다.

```python
s = {1, 2, 3}
empty = set()      # 빈 세트 (빈 {}는 dict임!)
```

### 15.1 특징

- 수학의 **집합**과 동일하게 동작한다.
- **중복 불가** → 똑같은 값은 단 하나만 남는다.
- **순서 없음** → 인덱싱·슬라이싱 사용 불가.

```python
s = {1, 1, 2, 2, 3}
print(s)   # {1, 2, 3}  ← 중복 자동 제거
```

### 15.2 집합 연산

```python
a = {1, 2, 3, 4}
b = {3, 4, 5, 6}

a | b    # 합집합 → {1, 2, 3, 4, 5, 6}
a & b    # 교집합 → {3, 4}
a - b    # 차집합 → {1, 2}
```

## 16. 그 외 타입 (Other Types)

### 16.1 None 타입

값이 **없음**을 표현하는 특별한 데이터 타입. (빈 상자)

```python
a = None
print(type(a))   # <class 'NoneType'>
```

> 다른 언어의 `null`에 해당하지만, 파이썬에는 `null`이 없고 **`None`** 만 있다.

### 16.2 불리언 (Boolean)

`True` / `False` 두 값만 갖는 타입. **온·오프 스위치**처럼 참/거짓을 판단하는 역할.

- 주로 **비교 연산**이나 **논리 연산**의 결과로 사용된다.

```python
3 > 2        # True
1 == 2       # False
True and False   # False
```

## 17. 컬렉션 (Collection) 정리

**컬렉션이란?** 여러 개의 값을 하나로 묶어 관리하는 자료형.
`str`, `list`, `tuple`, `range`, `set`, `dict`가 모두 컬렉션에 속한다.

### 17.1 컬렉션 비교표

| 컬렉션명 | 변경 가능 여부 (mutable) | 순서 존재 여부 |
|:--------:|:-----------------------:|:--------------:|
| `str`   | ❌ (immutable) | ✅ |
| `list`  | ✅ (mutable)   | ✅ |
| `tuple` | ❌ (immutable) | ✅ |
| `dict`  | ✅ (mutable)   | ❌ |
| `set`   | ✅ (mutable)   | ❌ |

### 17.2 핵심 — 불변(immutable) vs 가변(mutable)

- **불변(immutable):** `str`, `tuple`, `range` → 한 번 만들면 못 바꿈
- **가변(mutable):** `list`, `dict`, `set` → 자유롭게 수정·추가·삭제 가능

> **순서(order) ≠ 정렬(sorted):** 여기서 '순서 있음'은 값이 들어간 **차례가 있다**는 뜻이지, 자동으로 **정렬**된다는 뜻이 아니다.
