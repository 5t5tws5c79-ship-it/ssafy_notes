# Python 기초 정리 (Basic Syntax 1)

## 1. Python 인터프리터 실행법

터미널에서 `python` 명령어 뒤에 실행할 `.py` 파일의 경로를 입력한다.

```bash
python 01_fundamentals_of_python/01_basic.py
```

> **참고:** Git Bash에서는 경로 구분자로 백슬래시(`\`)가 잘 안 먹는 경우가 있다. 슬래시(`/`)를 사용하자.

## 2. 표현식 (Expression)

**표현식이란?** 하나의 값으로 평가(evaluate)될 수 있는 모든 코드.

```python
3 + 5      # 표현식
x > 10     # 표현식
```

---

## 3. 값 (Value)

**값이란?** 표현식이 평가된 결과. 더 이상 평가하거나 계산할 수 없는 데이터 조각.

- 종류: 숫자값, 문자열값, 불리언값 등
- `3 + 5`는 **표현식**이지만 그 자체로 값이 아니며, 평가를 거쳐야 값(`8`)이 된다.

| 구분 | 설명 | 예시 |
|------|------|------|
| 표현식 | 평가될 수 있는 코드 | `3 + 5` |
| 값 | 평가가 끝난 결과 | `8` |

---

## 4. 변수와 메모리 (Variable)

**변수란?** 값을 다시 사용하기 위해 저장해놓은 것.

### 변수 할당 (Assignment)

표현식이 만들어낸 값에 이름을 붙이는 과정. 이를 **할당문**이라 한다.

```python
degrees = 36.5
```

- 값 `36.5`를 변수 `degrees`에 할당했다.
- `=` 는 **할당 연산자**.

### 변수명 규칙

- 영문 알파벳, 언더스코어(`_`), 숫자로 구성
- 숫자로 시작할 수 없음
- 대소문자 구분 (`age` ≠ `Age`)
- 아래 **예약어(keyword)** 는 변수명으로 사용 불가
---

## 5. 메모리와 객체 (Object)

### 메모리 주소

값은 메모리 어딘가에 존재하며, 그 위치를 **메모리 주소**라 한다.
사물함 번호처럼 기계적으로 매겨진 숫자 주소.

### 객체 (Object)

**값 + 타입 + 주소 정보를 묶은 것**을 객체라고 부른다.
모든 객체는 고유한 **id / 타입 / 값**을 가진다.

### 변수 = 객체를 가리키는 이름표

변수는 특정 객체를 가리키는 **이름표(reference / point to)** 다.

> **비유 (주소록):**
> 주소록의 '내 친구 철수'라는 항목 자체가 실제 집 주소는 아니다.
> 하지만 그 이름을 보고 실제 집을 찾아갈 수 있다.
> `my_friend = '김철수'` 라고 쓰면, 변수 `my_friend`는 `'김철수'`라는 객체를 가리키는 이름표 역할을 한다.
> 파이썬은 이 이름표를 보고 실제 객체가 저장된 메모리 주소를 찾아 작업을 수행한다.

| 용어 | 핵심 정의 | 비유 (주소록) |
|------|-----------|---------------|
| 객체 (Object) | 데이터(값, 타입, 행동)의 실체 | '김철수'라는 실제 사람 |
| 메모리 주소 | 객체가 저장된 고유한 위치 | 김철수의 실제 집 주소 |
| 변수 (Variable) | 객체를 가리키는 이름표 | 주소록에 저장된 '내 친구 철수'라는 이름 |

## 6. 할당문 동작 순서

`variable = expression` 이 실행될 때:

1. **오른쪽 표현식 평가**
   할당 연산자(`=`)의 오른쪽 표현식 전체를 계산해 하나의 결과값(객체)을 만든다.

2. **왼쪽 변수명 확인**
   - 처음 사용된 이름이면 → 새로운 '이름표'를 준비
   - 이미 존재하는 이름이면 → 기존 '이름표'를 그대로 사용

3. **변수명과 결과값 연결 (참조)**
   왼쪽 변수명이 오른쪽에서 만들어진 결과값을 가리키도록 연결한다.
   기존에 다른 객체를 가리키고 있었다면 그 연결은 끊어지고, 새 객체와의 연결만 남는다.

### 재할당 (Reassignment)

변수를 새로운 값으로 변경하는 것.

```python
degrees = 36.5
degrees = 40.0   # 재할당 → 이제 40.0을 가리킴
```

## 7. 타입 (Type)

**데이터 타입이란?** 값의 종류와, 그 값으로 할 수 있는 동작을 정하는 지표.

### 타입이 필요한 이유

- 숫자는 더하기(`+` → 덧셈), 문자열은 결합(`+` → 이어붙이기)
- 숫자와 문자는 서로 더할 수 없음

```python
3 + 5          # 8       (숫자 덧셈)
"a" + "b"      # "ab"    (문자열 결합)
3 + "a"        # TypeError!
```

### 데이터 타입 종류 (5가지 분류)

| 분류 | 타입 |
|------|------|
| Numeric (숫자) | `int`, `float`, `complex` |
| Text (문자) | `str` |
| Sequence (순서 O) | `list`, `tuple`, `range` |
| Non-sequence (순서 X) | `set`, `dict` |
| 기타 | `bool`, `None`, `function` 등 |

> **참고:** 파이썬에는 C 언어 같은 `char` 타입이 **따로 없다.** 문자 하나도 길이 1짜리 `str`로 취급한다.

## 8. 정수형 (Integer, int)

정수를 표현하는 자료형. 소수점 없는 숫자.

```python
age = 25
count = -3
big = 1_000_000   # 언더스코어로 자릿수 구분 가능
```

## 9. 실수형 (Float)

소수점이 있는 숫자를 표현하는 자료형.

```python
pi = 3.14
temp = -0.5
```

### 지수 표현 (Exponential)

매우 크거나 작은 수를 `e`(또는 `E`)를 써서 표현한다. `e` 뒤의 숫자는 10의 거듭제곱.

```python
a = 3e2      # 3 × 10^2  = 300.0
b = 1.5e-3   # 1.5 × 10^-3 = 0.0015
```

## 10. 산술 연산자 (Arithmetic Operators)

| 연산자 | 의미 | 예시 | 결과 |
|--------|------|------|------|
| `+` | 덧셈 | `5 + 2` | `7` |
| `-` | 뺄셈 | `5 - 2` | `3` |
| `*` | 곱셈 | `5 * 2` | `10` |
| `/` | 나눗셈 (항상 float) | `5 / 2` | `2.5` |
| `//` | 몫 (정수 나눗셈) | `5 // 2` | `2` |
| `%` | 나머지 (modulo) | `5 % 2` | `1` |
| `**` | 거듭제곱 | `5 ** 2` | `25` |

### 연산자 우선순위 (Operator Precedence)

파이썬도 수학처럼 정해진 순서에 따라 연산을 수행한다.

```python
-2 ** 4    # 결과: -16
```

- `**`(거듭제곱)가 단항 `-`(부호)보다 **먼저** 계산되기 때문에 `-(2 ** 4)` = `-16`이 된다.
- 헷갈리기 쉬우므로 **괄호로 명확하게 표현**하는 것이 중요하다.

```python
(-2) ** 4  # 결과: 16
```

## 11. 시퀀스 타입 (Sequence Type)

**순서(order)가 있는** 자료형. (여기서 '순서'는 값이 들어간 차례가 있다는 뜻이지, **정렬(sorted)** 되어 있다는 뜻이 아니다.)

### 시퀀스의 공통 기능

- **인덱싱 (Indexing)** — 특정 위치의 값 꺼내기
- **슬라이싱 (Slicing)** — 일정 범위 잘라내기
- **길이 (Length)** — `len()`으로 개수 확인
- **반복 (Iteration)** — 반복문으로 각 값을 하나씩 순서대로 꺼내 사용

```python
my_data = 'hello'   # 문자열도 사실 시퀀스다!

my_data[0]          # 'h'  (인덱싱)
my_data[1:3]        # 'el' (슬라이싱)
len(my_data)        # 5    (길이)

for char in my_data:  # 반복
    print(char)
```

## 12. 문자열 (str)

**문자들의 순서가 있는, 변경 불가능한(immutable) 시퀀스** 자료형.

### 문자열 만들기 — 따옴표 활용

```python
a = 'hello'
b = "hello"
```

- 작은따옴표(`'`)와 큰따옴표(`"`)는 기능상 같다.
- 한 프로젝트 안에서 **혼용하지 않는 것을 권장**한다.

### 문자열 안에 따옴표 넣기

서로 **다른 종류**의 따옴표를 사용한다.

```python
msg = "It's a book"      # 작은따옴표를 문자열 안에
quote = 'He said "hi"'   # 큰따옴표를 문자열 안에
```

### 이스케이프 시퀀스 (Escape Sequence)

백슬래시(`\`)로 특수 문자를 표현한다.

| 이스케이프 | 의미 |
|-----------|------|
| `\n` | 줄바꿈 (new line) |
| `\t` | 탭 |
| `\\` | 백슬래시 문자 그대로 |
| `\'` | 작은따옴표 |
| `\"` | 큰따옴표 |

### 여러 줄 문자열

삼중 따옴표(`"""` 또는 `'''`)를 사용한다.

```python
text = """여러 줄
문자열을
쓸 수 있다"""
```

### f-string (formatted string)

문자열 앞에 `f`를 붙이고 `{}` 안에 변수나 표현식을 넣으면, 그 값이 문자열에 삽입된다.

```python
name = 'Alice'
age = 25

print(f'제 이름은 {name}이고 {age}살입니다.')
# 제 이름은 Alice이고 25살입니다.

print(f'내년엔 {age + 1}살')   # {} 안에 표현식도 가능 → 내년엔 26살
```

## 13. 인덱스 (Index)

### 왜 인덱스는 0부터 시작할까?

**거리(offset)의 개념**으로 이해하면 쉽다. 인덱스는 "시작점에서 얼마나 떨어져 있는가"를 뜻한다. 맨 앞 요소는 시작점에서 **0만큼** 떨어져 있으므로 인덱스가 `0`이다.

### `'hello'`의 인덱스 표

파이썬은 뒤에서부터 세는 **음수 인덱스**도 지원한다.

| 문자 | `h` | `e` | `l` | `l` | `o` |
|------|-----|-----|-----|-----|-----|
| 양수 인덱스 | 0 | 1 | 2 | 3 | 4 |
| 음수 인덱스 | -5 | -4 | -3 | -2 | -1 |

```python
my_str = 'hello'
my_str[0]    # 'h'
my_str[-1]   # 'o'  (맨 뒤)
```

## 14. 슬라이싱 (Slicing)

일정 범위를 잘라내는 기능. `start`는 포함, `stop`은 **미포함**.

```python
my_sequence[start:stop:step]
```

세 값 모두 상황에 따라 생략할 수 있다.

```python
my_str = 'hello'

my_str[2:4]    # 'll'   (인덱스 2 ~ 3)
my_str[:3]     # 'hel'  (처음 ~ 인덱스 2)
my_str[::2]    # 'hlo'  (처음부터 끝까지 2칸씩)
my_str[::-1]   # 'olleh' (거꾸로! 파이썬의 강력한 기능)
```

| 구성 요소 | 의미 | 생략 시 기본값 |
|-----------|------|----------------|
| `start` | 시작 인덱스 (포함) | 처음 (0) |
| `stop` | 끝 인덱스 (미포함) | 끝까지 |
| `step` | 건너뛰는 간격 | 1 |

## 15. 문자열은 변경 불가능 (Immutable)

문자열은 **변경 불가능한 시퀀스**라서, 일부 문자만 바꾸려 하면 에러가 난다.

```python
my_str = 'hello'
my_str[1] = 'a'   # ❌ TypeError!
```

문자열을 "바꾸려면" 새로운 문자열을 **새로 만들어** 재할당해야 한다. (기존 객체를 수정하는 것이 아니라, 완전히 새 객체를 만드는 것 — 재할당과 헷갈리면 안 된다.)

```python
my_str = 'hello'
my_str = 'hallo'          # 새 문자열 객체로 재할당
# 또는
my_str = 'h' + 'a' + my_str[2:]   # 'hallo'
```

> **핵심:** `my_str[1] = 'a'`(내부 수정)는 불가능하지만, `my_str = 새로운문자열`(재할당)은 가능하다. 전자는 원본을 바꾸려는 시도(불가), 후자는 이름표를 새 객체로 옮기는 것(가능)이라는 점이 다르다.
