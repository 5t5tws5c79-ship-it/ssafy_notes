# ✍️ 마크다운(Markdown) 기초

## 1. 마크다운이란?

텍스트에 **간단한 기호**만 붙여 문서 구조와 서식을 표현하는 **마크업 언어(Markup Language)**.

- 복잡한 태그 없이 `#`, `*`, `-` 같은 기호로 서식을 지정
- 확장자는 `.md`

### 왜 개발자들이 쓸까?

- **간단함** — 배우기 쉽고 빠르게 작성
- **가독성** — 기호가 있어도 원문 그대로 읽기 편함
- **호환성** — GitHub, 블로그, 노션 등 어디서나 렌더링됨
- README, 문서, 기술 블로그의 사실상 표준

## 2. 구조 문법

### 제목 (Heading)

`#` 개수로 `h1`~`h6`을 표현한다. (`#`이 많을수록 작은 제목)

```markdown
# 제목 1 (h1)
## 제목 2 (h2)
### 제목 3 (h3)
#### 제목 4 (h4)
##### 제목 5 (h5)
###### 제목 6 (h6)
```

> 💡 `h1`은 문서당 **하나만** 쓰는 것을 추천 (문서의 대표 제목)

### 목록 (List)

**순서 없는 목록** — `-`, `*`, `+`

```markdown
- 사과
- 바나나
  - 하위 항목 (탭 또는 스페이스 2번으로 들여쓰기)
```

**순서 있는 목록** — `숫자.`

```markdown
1. 첫 번째
2. 두 번째
3. 세 번째
```

> 번호는 `1. 1. 1.`로 모두 1을 써도 자동으로 1, 2, 3 순서가 매겨진다.

## 3. 컨텐츠 문법

### 텍스트 강조

```markdown
**굵게 (Bold)**
*기울임 (Italic)*
~~취소선 (Strikethrough)~~
```

→ **굵게**, *기울임*, ~~취소선~~

### 수평선

`---`로 구분선을 긋는다.

```markdown
---
```

### 인용문

`>`로 문장을 인용한다.

```markdown
> 인용할 내용을 여기에 작성합니다.
```

### 링크와 이미지

**링크** — `[텍스트](URL)`

```markdown
[구글](https://google.com)
```

**이미지** — 링크 앞에 `!`가 붙는다. `![대체텍스트](경로)`

```markdown
![고양이 사진](https://example.com/cat.png)   <!-- 온라인 이미지 -->
![로고](./images/logo.png)                   <!-- 로컬 이미지 -->
```

> 대괄호 `[ ]`에는 설명(대체 텍스트), 소괄호 `( )`에는 경로/URL이 들어간다.  
> 이미지 경로는 웹 주소(온라인)와 폴더 경로(로컬) 둘 다 사용 가능.  
> 로컬 경로 구분자는 `/`를 쓰며, Windows에서도 그대로 동작한다. (`\` 대신 `/` 사용 권장)

### 코드 (Code)

백틱(`` ` ``)으로 코드를 표현한다.

**인라인 코드** — 백틱 하나로 감싼다.

```markdown
`git init` 명령어를 실행합니다.
```

→ `git init` 명령어를 실행합니다.

**코드 블록** — 백틱 세 개로 감싸고, 뒤에 언어를 적으면 문법 강조가 된다.

````markdown
```bash
git add .
git commit -m "메시지"
```
````

## 4. 참고: 마크다운은 표준이 하나가 아니다

마크다운은 **공식(오피셜) 표준 문법이 정해져 있지 않아**, 사용하는 곳마다 문법이 조금씩 다르고 커스터마이징이 많다. (GitHub Flavored Markdown, 노션 등)

그래서 기본 문법을 참고할 때는 아래 가이드를 보는 것이 좋다.

- 기본 문법(Basic Syntax): <https://www.markdownguide.org/getting-started/>
