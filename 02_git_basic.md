# 🛠️ Git 핵심 기초 정리

## 1. 버전 관리란?

파일 변경 사항을 시간순으로 기록·관리하는 시스템.

- **이력 추적** — 누가, 언제, 무엇을 바꿨는지 확인
- **되돌리기** — 문제 발생 시 이전 버전으로 복원
- **협업** — 여러 명이 동시에 작업해도 안전하게 병합

> Git은 2005년 리누스 토르발즈가 만든 **분산형** 버전 관리 도구. 빠른 속도와 강력한 브랜치·병합 기능이 특징.

---

## 2. Git의 3가지 영역

| 영역 | 역할 |
|------|------|
| **Working Directory** | 실제로 파일을 만들고 수정하는 작업 공간 (프로젝트 폴더) |
| **Staging Area** | 커밋할 파일을 골라 담는 임시 공간 (장바구니) |
| **Repository** | 커밋된 버전이 영구 저장되는 공간 |

```
Working Directory ──git add──▶ Staging Area ──git commit──▶ Repository
```

---

## 3. 커밋 = 버전 스냅샷

Git은 차이점이 아니라 **그 시점의 프로젝트 상태 전체**를 스냅샷으로 저장한다.

> **커밋 1개 = 완성된 버전 1개** → 언제든 되돌아갈 수 있음

```
Commit 1 회원가입 화면 → Commit 2 로그인 추가 → Commit 3 회원정보 수정
```

---

## 4. 기본 명령어 4가지

| 명령어 | 설명 |
|--------|------|
| `git init` | 현재 폴더를 Git 저장소로 초기화 (`.git` 생성) |
| `git status` | 현재 상태 확인 (수정·추가·삭제·스테이징 여부) |
| `git add` | 변경 파일을 스테이징 영역에 추가 |
| `git commit` | 스테이징된 변경을 하나의 버전으로 저장 |

```bash
git init                        # 저장소 시작
git status                      # 상태 확인 (가장 자주 사용)
git add index.html             # 특정 파일 추가
git add .                       # 전체 변경 추가
git commit -m "회원가입 기능 추가"   # 버전 저장
```

---

## 5. 좋은 커밋 메시지

- 무엇을 바꿨는지 **간결·명확**하게
- **한 커밋 = 한 가지 작업**

```bash
git commit -m "로그인 오류 수정"
git commit -m "README 작성"
```

---

## 6. `git init` 주의사항

- **저장소 안에 저장소(중첩) 금지**  
  `git init`을 하면 그 폴더 **하위의 모든 파일**이 하나의 저장소로 관리된다.  
  Git은 **저장소 안의 또 다른 Git 저장소를 제대로 인식하지 못하므로**, 이미 Git으로 관리 중인 폴더 안에서 또 `git init`을 하지 말 것.

- **`.git` 폴더란?**  
  `git init` 시 생기는 숨김 폴더로, 커밋 이력·버전 정보 등 저장소의 모든 데이터가 들어 있다. (사실상 이 폴더가 곧 저장소)

- **`git init` 취소(복구)**  
  잘못 초기화했다면 `.git` 폴더를 지우면 된다.

  ```bash
  rm -rf .git
  ```
